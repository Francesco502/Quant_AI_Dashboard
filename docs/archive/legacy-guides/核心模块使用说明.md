## 🛡️ 风险管理模块

### 模块概览

风险管理模块包含以下组件：

1. **风险类型定义** (`risk_types.py`) - 风险等级、动作、告警严重程度等枚举
2. **仓位管理器** (`position_manager.py`) - 多层级仓位限制
3. **止损止盈管理器** (`stop_loss_manager.py`) - 止损止盈规则管理
4. **风险监控器** (`risk_monitor.py`) - 实时风险监控
5. **风险告警系统** (`risk_alerting.py`) - 多通道告警

### 快速开始

```python
from core.risk_types import RiskLimits, RiskAction
from core.risk_monitor import RiskMonitor
from core.position_manager import PositionManager, SectorInfo
from core.stop_loss_manager import StopLossManager

# 创建风险限制配置
risk_limits = RiskLimits(
    max_position_size=0.1,      # 单仓位最大10%
    max_single_stock=0.05,      # 单股票最大5%
    max_daily_loss=0.05,        # 单日最大亏损5%
    max_total_loss=0.2,         # 总亏损最大20%
)

# 创建仓位管理器
sector_info = {
    "AAPL": SectorInfo(symbol="AAPL", sector="科技", market="美股"),
}
position_manager = PositionManager(sector_info=sector_info)

# 创建风险监控器
risk_monitor = RiskMonitor(
    risk_limits=risk_limits,
    position_manager=position_manager
)

# 检查订单风险
order = {
    "symbol": "AAPL",
    "side": "BUY",
    "quantity": 100,
    "price": 150.0
}
result = risk_monitor.check_order_risk(order, portfolio, current_prices)

if result.action == RiskAction.ALLOW:
    print("订单通过风险检查")
else:
    print(f"订单被拒绝: {result.message}")
```

### 仓位管理

```python

    symbol="AAPL",
    quantity=100,
    portfolio=portfolio,
    current_prices=current_prices

# 获取可用仓位
available = position_manager.get_available_position(
    symbol="AAPL",
    portfolio=portfolio,
    current_prices=current_prices
)
```

### 止损止盈

```python
    current_prices={"AAPL": 140.0},  # 触发止损
    portfolio=portfolio
)
```

## 🔍 全市场扫描 (Market Scanner)

### 模块概览

全市场扫描器 (`core/market_scanner.py`) 用于在成千上万只股票中快速筛选符合特定策略条件的标的。

### 使用方法

**代码调用**:
```python
from core.market_scanner import MarketScanner

scanner = MarketScanner()

# 定义策略配置
config = {
    "name": "BBI_KDJ",
    "params": {
        "j_threshold": -5,
        "bbi_min_window": 90
    }
}

# 执行扫描 (A股市场)
results = scanner.scan_market(
    strategy_config=config,
    market="CN",
    batch_size=50,
    limit=200  # 限制数量用于测试
)

for res in results:
    print(f"发现标的: {res['ticker']} 价格: {res['price']}")
```

**前端使用**:
进入 Dashboard 的 "交易中心" -> "市场扫描" 标签页，点击 "开始扫描" 即可。

## 📈 策略回测 (Backtesting)

### 模块概览

回测系统 (`core/backtest.py`) 允许用户在历史数据上验证策略的有效性。

### 使用方法

**代码调用**:
```python
from api.routers.backtest import run_backtest, BacktestRequest

req = BacktestRequest(
    strategy_id="sma_crossover",
    tickers=["000001.SZ", "600519.SH"],
    start_date="2023-01-01",
    end_date="2023-12-31",
    initial_capital=100000.0,
    params={"short_window": 10, "long_window": 30}
)

# 运行回测
result = await run_backtest(req)
print(f"最终权益: {result.final_equity}")
print(f"总收益率: {result.total_return * 100}%")
```

**前端使用**:
进入 Dashboard 的 "策略回测" 页面，配置参数并运行，系统将自动绘制权益曲线。

## 🤖 后台守护进程 (Daemon)

### 模块概览

守护进程 (`core/daemon.py`) 是系统的"心脏"，负责维持系统的自动化运行。它基于 `schedule` 库运行定时任务。

### 任务列表

1.  **数据更新**: 每日收盘后自动从数据源拉取最新行情并更新本地缓存。
2.  **模型训练**: 定期（如每周）重新训练 AI 模型以适应市场变化。
3.  **自动交易**: 根据配置的策略，在指定时间自动生成信号并执行模拟交易。

### 启动方式

```bash
python run_daemon.py
```

配置文件位于项目根目录的 `daemon_config.json` (首次运行自动生成)。

### 模块概览

交易执行系统包含以下组件：

1. **订单类型定义** (`order_types.py`) - 订单类型、方向、状态等枚举
2. **订单管理器** (`order_manager.py`) - 完整的订单生命周期管理
3. **滑点模型** (`slippage_model.py`) - 固定、基于成交量、基于波动率的滑点
4. **执行算法** (`execution_algorithms.py`) - TWAP、VWAP、自适应算法

### 快速开始

```python
from core.order_manager import OrderManager
from core.order_types import OrderType, OrderSide
from core.slippage_model import SlippageModel, SlippageConfig
from core.execution_algorithms import get_execution_algorithm

# 创建订单管理器
order_manager = OrderManager()

# 创建滑点模型
slippage_config = SlippageConfig(model_type="fixed", fixed_rate=0.001)
slippage_model = SlippageModel(slippage_config)

# 创建执行算法
execution_algorithm = get_execution_algorithm("twap", duration_minutes=30, num_slices=10)

# 创建订单
order = order_manager.create_order(
    symbol="AAPL",
    side=OrderSide.BUY,
    quantity=100,
    order_type=OrderType.MARKET,
)

# 提交订单
order_manager.submit_order(order.order_id)

# 执行订单
fills = execution_algorithm.execute(
    order=order,
    current_price=150.0,
    market_data=market_data,
)

# 应用滑点并记录成交
for fill in fills:
    adjusted_price = slippage_model.apply_slippage(order, fill.price)
    fill.price = adjusted_price
    order_manager.add_fill(order.order_id, fill)
```

### 滑点模型

```python
from core.slippage_model import SlippageModel, SlippageConfig

# 固定滑点模型
config = SlippageConfig(model_type="fixed", fixed_rate=0.001)  # 0.1%滑点
slippage_model = SlippageModel(config)

# 基于成交量的滑点模型
config = SlippageConfig(
    model_type="volume",
    fixed_rate=0.001,
    volume_impact_factor=0.5
)
slippage_model = SlippageModel(config)

# 计算滑点
slippage = slippage_model.calculate_slippage(order, current_price=150.0)
adjusted_price = slippage_model.apply_slippage(order, current_price=150.0)
```

### 执行算法

```python
from core.execution_algorithms import get_execution_algorithm

# TWAP算法
twap = get_execution_algorithm("twap", duration_minutes=30, num_slices=10)
fills = twap.execute(order, current_price=150.0)

# VWAP算法
vwap = get_execution_algorithm("vwap", lookback_days=20, num_slices=10)
fills = vwap.execute(order, current_price=150.0, market_data=market_data)

# 自适应算法（自动选择）
adaptive = get_execution_algorithm("adaptive", large_order_threshold=0.1)
fills = adaptive.execute(order, current_price=150.0, market_data=market_data)
```

## 📊 监控与告警系统

### 模块概览

监控与告警系统包含以下组件：

1. **指标收集器** (`monitoring/metrics.py`) - 收集和存储系统指标
2. **健康检查器** (`monitoring/health_checker.py`) - 健康检查
3. **系统监控器** (`monitoring/system_monitor.py`) - 系统指标收集
4. **告警管理器** (`monitoring/alert_manager.py`) - 告警规则管理

### 快速开始

```python
from core.monitoring import (
    SystemMonitor,
    MetricsCollector,
    HealthChecker,
    AlertManager,
    AlertSeverity,
    ComparisonOperator,
)

# 创建组件
metrics_collector = MetricsCollector()
health_checker = HealthChecker()
alert_manager = AlertManager(metrics_collector=metrics_collector)

system_monitor = SystemMonitor(
    metrics_collector=metrics_collector,
    health_checker=health_checker,
    collection_interval=60.0,  # 每60秒收集一次
)

# 启动监控
system_monitor.start_monitoring()

# 添加告警规则
alert_manager.add_alert_rule(
    name="CPU告警",
    metric_name="cpu_usage",
    threshold=80.0,
    comparison=ComparisonOperator.GT,
    severity=AlertSeverity.WARNING,
    channels=["dashboard", "email"],
)

# 检查并触发告警
system_monitor.collect_metrics()
alert_manager.check_and_trigger()
```

### 指标收集

```python
from core.monitoring import MetricsCollector

collector = MetricsCollector()

# 记录指标
collector.record({
    "cpu_usage": 50.0,
    "memory_usage": 60.0,
    "disk_usage": 70.0,
})

# 获取最新指标
latest_cpu = collector.get_latest_metric("cpu_usage")

# 获取指标统计
stats = collector.get_metric_statistics("cpu_usage", window_minutes=60)
print(f"平均值: {stats['mean']:.2f}")
print(f"最大值: {stats['max']:.2f}")
```

### 健康检查

```python
from core.monitoring import HealthChecker

health_checker = HealthChecker()

# 执行所有健康检查
checks = health_checker.check_all()

# 获取整体状态
overall_status = health_checker.get_overall_status(checks)
print(f"整体状态: {overall_status.value}")

# 单独检查
disk_check = health_checker.check_disk_space(threshold_percent=90.0)
memory_check = health_checker.check_memory(threshold_percent=90.0)
```

### 告警管理

```python
from core.monitoring import AlertManager, EmailChannel, WebhookChannel

alert_manager = AlertManager()

# 配置告警渠道
email_channel = EmailChannel(
    smtp_server="SMTP_HOST",
    smtp_port=587,
    username="SMTP_USERNAME",
    password="SMTP_PASSWORD_FROM_ENV",
)
alert_manager.add_channel("email", email_channel)

webhook_channel = WebhookChannel(webhook_url="https://your-webhook-url.com")
alert_manager.add_channel("webhook", webhook_channel)

# 添加告警规则
rule_id = alert_manager.add_alert_rule(
    name="内存告警",
    metric_name="memory_usage",
    threshold=90.0,
    comparison=ComparisonOperator.GT,
    severity=AlertSeverity.CRITICAL,
    channels=["email", "webhook"],
    cooldown_minutes=10,
)

# 检查并触发告警
metrics = {"memory_usage": 95.0}
alert_manager.check_and_trigger(metrics)

# 查看告警历史
history = alert_manager.get_alert_history(limit=100)
```

```python
from core.data_service import load_price_data, load_ohlcv_data

# 获取价格数据
data = load_price_data(

    ticker="159755.SZ",
```

```python
from core.data_validation import DataValidator

validator = DataValidator()

# 验证数据
result = validator.validate(data, ticker="159755.SZ")

if result.is_valid:
    print("数据验证通过")
else:
    print(f"发现 {len(result.issues)} 个问题:")
    for issue in result.issues:
        print(f"  - {issue.type}: {issue.message}")
```
