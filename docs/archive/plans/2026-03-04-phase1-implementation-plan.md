## 任务清单

### 任务 1: 增强模拟交易前端 - 权益曲线与绩效分析

**目标**: 在 Trading 页面添加实时权益曲线图和绩效指标卡片

**文件变更**:
```
api/routers/trading.py          # 新增权益历史/绩效API端点
web/src/app/trading/page.tsx    # 新增图表和指标展示
web/src/lib/api.ts              # 新增API接口定义
```

**详细步骤**:

1. **后端API增强** (`api/routers/trading.py`)
   - 新增 `GET /api/trading/paper/performance` - 获取账户绩效指标
     - 返回：总收益率、年化收益率、夏普比率、最大回撤、胜率
   - 新增 `GET /api/trading/paper/equity-curve` - 获取实时权益曲线数据
     - 基于当前持仓市值 + 现金计算历史权益点
     - 返回时间序列数据用于图表

2. **前端页面增强** (`web/src/app/trading/page.tsx`)
   - 在"模拟账户"Tab顶部添加绩效指标卡片行（4列布局）:
     - 累计收益率（红涨绿跌）
     - 夏普比率
     - 最大回撤
     - 胜率
   - 添加权益曲线图区域:
     - 使用 Recharts AreaChart
     - 支持时间范围切换（7天/30天/90天/全部）
     - 显示总资产走势

3. **API客户端更新** (`web/src/lib/api.ts`)
   - 新增接口：`PerformanceMetrics`, `EquityPoint`
   - 新增方法：`api.trading.paper.getPerformance()`, `api.trading.paper.getEquityCurve()`

**验收标准**:
- [ ] 打开 Trading 页面能看到权益曲线图
- [ ] 绩效指标卡片显示正确数据
- [ ] 时间范围切换能更新图表数据

### 任务 2: 增强选股扫描功能

**目标**: 在 Trading 页面的"市场扫描"Tab中增强策略选择和结果展示

**文件变更**:
```
api/routers/scanner.py              # 扩展API支持更多参数
web/src/app/trading/page.tsx        # 增强扫描配置和结果展示
web/src/lib/api.ts                  # 更新扫描相关接口
```

**详细步骤**:

1. **后端API扩展** (`api/routers/scanner.py`)
   - 扩展 `POST /api/scanner/select`:
     - 支持 `market` 参数："CN"/"HK"/"WATCHLIST"
     - 支持 `min_score` 参数：最低得分阈值
     - 支持 `strategy` 参数：单策略或"all"
   - 确保 `GET /api/scanner/strategies` 返回完整的策略列表和参数说明

2. **前端页面增强** (`web/src/app/trading/page.tsx`)
   - 在扫描配置面板添加：
     - 市场选择：A股/港股/自选股
     - 策略参数配置：根据策略显示可调参数
     - 最小分数滑块：30-100范围
   - 增强结果展示：
     - 显示股票得分
     - 显示信号类型（买入/观望/卖出）
     - 添加"查看决策"按钮，可跳转到决策分析
     - 添加"添加到自选股"按钮

3. **API客户端更新** (`web/src/lib/api.ts`)
   - 更新 `ScanMarketRequest` 接口
   - 更新 `ScanResultItem` 接口包含 score 和 signal

**验收标准**:
- [ ] 可以选择不同市场（A股/港股）进行扫描
- [ ] 策略参数可以配置
- [ ] 扫描结果显示得分和信号
- [ ] 可以点击查看决策分析

### 任务 3: 在持仓分析页面集成决策仪表盘

**目标**: 在 `/portfolio-analysis` 页面为每个持仓显示决策仪表盘卡片

**文件变更**:
```
web/src/app/portfolio-analysis/page.tsx   # 集成决策仪表盘展示
web/src/components/trading/               # 新增决策卡片组件
```

**详细步骤**:

1. **创建决策卡片组件** (`web/src/components/trading/DecisionCard.tsx`)
   - 组件Props: `ticker: string`, `compact?: boolean`
   - 展示内容：
     - 顶部：股票代码 + 当前操作建议（买入/观望/卖出）+ 得分
     - 中部：一句话结论
     - 底部：关键检查清单（3-4项）
   - 紧凑模式：只显示操作建议和得分
   - 完整模式：展开显示买卖点、完整检查清单、亮点和风险

2. **后端API确认** (`api/routers/portfolio.py`)
   - 确认 `GET /api/portfolio/decision/{ticker}` 可用
   - 返回数据结构：
     ```json
     {
       "ticker": "600519",
       "conclusion": "一句话结论...",
       "action": "观望",
       "score": 55,
       "buy_price": null,
       "stop_loss": null,
       "target_price": null,
       "checklist": [...],
       "highlights": [...],
       "risks": [...]
     }
     ```

3. **前端页面更新** (`web/src/app/portfolio-analysis/page.tsx`)
   - 在持仓输入区域添加"自动获取决策"选项
   - 分析结果区域为每个持仓显示决策卡片（紧凑模式）
   - 点击卡片可展开查看完整决策详情
   - 添加批量获取所有持仓决策的按钮

4. **API客户端更新** (`web/src/lib/api.ts`)
   - 新增接口：`DecisionDashboardResult`
   - 新增方法：`api.portfolio.getDecision(ticker: string)`

**验收标准**:
- [ ] 输入持仓后可以获取每个股票的决策分析
- [ ] 决策卡片显示操作建议和得分
- [ ] 可以展开查看完整的买卖点、检查清单
- [ ] 亮点和风险提示清晰展示

## 风险与注意事项

1. **性能风险**：权益曲线实时计算可能较慢（需要遍历历史持仓）
   - 缓解：首次加载后缓存结果，只计算最新一天

2. **数据准确性**：决策仪表盘基于历史数据计算，可能存在延迟
   - 缓解：显示数据更新时间戳

3. **API兼容性**：需要确保后端API返回格式与前端期望一致
   - 缓解：实施前进行API测试
