# 2.2.1 Scheme A Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the first vertical slice of the 2.2.1 personal quant workflow: daily decision workbench, data freshness checks, audit trail, backups, and tighter front/back API contracts.

**Architecture:** Keep heavy quantitative logic in existing services. Add thin service and router layers for freshness, audit, backup, and daily snapshot aggregation, then expose typed frontend APIs and a dedicated workbench page. Wire critical user actions to the new preflight/audit layer without rewriting the whole product.

**Tech Stack:** FastAPI, pytest, Next.js App Router, TypeScript, shared UI cards/tables.
