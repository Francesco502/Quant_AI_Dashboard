## 一、项目目标重新定位

### 1.1 原目标 vs 新目标

| 维度 | 原目标（生产级量化交易系统） | 新目标（个人量化分析学习系统） |
|------|---------------------------|------------------------------|
| **定位** | 面向机构/专业交易者的生产系统 | 面向个人投资者的学习分析工具 |
| **核心价值** | 自动化交易执行 | 分析辅助、策略验证、学习实践 |
| **交易方式** | 支持实盘/模拟自动化交易 | 仅模拟交易，实际购买手动执行 |
| **时间粒度** | 支持分钟线/Tick级别 | **仅日线级别**，专注长期分析 |
| **部署目标** | 高性能服务器 | **2核2GB低配服务器** |
| **用户规模** | 多用户并发 | 少量用户，数据隔离 |

### 1.2 新目标的8大核心需求

| 编号 | 需求描述 | 优先级 | 备注 |
|------|----------|--------|------|
| 1 | 个人指定资产分析（股票、基金、黄金），分析风险、涨跌 | 🔴 高 | 需要直观的决策摘要 |
| 2 | 全市场（A股、港股）找股，按策略筛选优质股票 | 🔴 高 | 购买手动执行 |
| 3 | 模拟账户量化交易，验证策略，观察盈亏 | 🔴 高 | 按量化方式进行 |
| 4 | 回测功能 | 🔴 高 | 策略历史验证 |
| 5 | 多用户登录，各用户独立配置 | 🟡 中 | 资产配置、策略配置 |
| 6 | 日线级别长期分析 | 🔴 高 | **不需要**分钟线/分线 |
| 7 | 2核2GB服务器Docker部署 | 🔴 高 | 性能优化适配 |
| 8 | 借鉴成熟项目优化 | 🟡 中 | daily_stock_analysis、dexter |

## 二、需求与现状差距分析

### 2.1 需求1：个人指定资产分析

#### 现状评估
| 功能 | 现状 | 状态 |

#### 存在差距
| 差距项 | 影响 | 严重度 |
|--------|------|--------|
| **缺少"我的持仓"统一入口** | 用户无法批量管理和分析持仓 | 🔴 高 |

   - 集成 AkShare 黄金数据接口（`ak.gold_spot_em()`）

3. **实现决策仪表盘**

   - 输出：一句话结论 + 精确买卖点位 + 操作检查清单

#### 4.2.1 数据库 schema 更新

```sql

CREATE TABLE IF NOT EXISTS user_watchlist (

    ticker TEXT NOT NULL,
```

```python
import importlib
from functools import lru_cache

class LazyModule:
    def __init__(self, module_name):
        self._module_name = module_name
        self._module = None

    def __getattr__(self, name):
        if self._module is None:
            self._module = importlib.import_module(self._module_name)
        return getattr(self._module, name)
```

#### 6.1.2 缓存策略优化

修改 `core/multi_level_cache.py`:
```python
from functools import lru_cache
from collections import OrderedDict
import time

class LRUMemoryCache:
    def __init__(self, maxsize=100, ttl=3600):
        self.cache = OrderedDict()
        self.maxsize = maxsize
        self.ttl = ttl

    def get(self, key):
        if key not in self.cache:
            return None
        value, timestamp = self.cache[key]
        if time.time() - timestamp > self.ttl:
            del self.cache[key]
            return None
        self.cache.move_to_end(key)
        return value

    def put(self, key, value):
        if len(self.cache) >= self.maxsize:
            self.cache.popitem(last=False)
        self.cache[key] = (value, time.time())
```

#### 6.1.3 内存监控告警

创建 `core/memory_monitor.py`:
```python
import psutil
import logging
import threading
import time

logger = logging.getLogger(__name__)

class MemoryMonitor:
    def __init__(self, warning_threshold=0.8, critical_threshold=0.9):
        self.warning_threshold = warning_threshold
        self.critical_threshold = critical_threshold
        self._running = False
        self._thread = None

    def get_memory_usage(self):
        return psutil.virtual_memory().percent / 100

    def _monitor_loop(self):
        while self._running:
            usage = self.get_memory_usage()
            if usage >= self.critical_threshold:
                logger.critical(f"内存使用率严重: {usage:.1%}")
            elif usage >= self.warning_threshold:
                logger.warning(f"内存使用率偏高: {usage:.1%}")
            time.sleep(60)

    def start(self):
        if not self._running:
            self._running = True
            self._thread = threading.Thread(target=self._monitor_loop, daemon=True)
            self._thread.start()

    def stop(self):
        self._running = False
        if self._thread:
            self._thread.join()
```
