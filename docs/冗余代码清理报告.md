## 详细清理内容

### 1. LSTM/GRU深度学习模型

**修改文件**: `core/advanced_forecasting.py`

**修改内容**:
- `LSTMForecaster` 类修改为初始化时抛出 `RuntimeError`
- `GRUForecaster` 类修改为初始化时抛出 `RuntimeError`
- 保留原始类定义作为文档参考

**原因**:
- 2GB内存环境下无法正常运行PyTorch深度学习模型
- 个人用户推荐使用轻量级模型：XGBoost、LightGBM、Prophet、RandomForest

**影响范围**:
- 预测功能仍可用，使用轻量级模型替代
- 错误信息清晰指引用户使用替代方案

### 2. 实盘交易接口

**删除文件**: `core/brokers/mock_real_broker.py`

**原因**:
- 项目仅需要模拟交易功能
- `paper_broker.py` 和 `backtest_broker.py` 已满足需求

**保留文件**:
- `core/brokers/backtest_broker.py` - 回测Broker
- `core/brokers/paper_broker.py` - 模拟交易Broker

---

### 3. 复杂执行算法(TWAP/VWAP)

**修改文件**: `core/execution_algorithms.py`

**修改前**: 399行代码，包含：
- `ExecutionAlgorithm` 基类
- `MarketOrderAlgorithm` 市价单
- `TWAPAlgorithm` 时间加权平均价格
- `VWAPAlgorithm` 成交量加权平均价格
- `AdaptiveAlgorithm` 自适应算法

**修改后**: ~60行代码，仅保留：
- `ExecutionAlgorithm` 简化版（市价单执行）
- `get_execution_algorithm()` 函数

**原因**:
- TWAP/VWAP是机构级分批执行算法
- 个人用户无需分批执行，市价单即可满足需求

**代码量减少**: 约340行 (85%)

---

### 4. RBAC权限系统

**修改文件**: `core/rbac.py`

**修改前**: 201行代码，包含：
- 4个角色: ADMIN, TRADER, VIEWER, ANALYST
- 9个权限
- 复杂的权限管理方法

**修改后**: ~110行代码，包含：
- 2个角色: ADMIN, USER
- 7个权限
- 简化的权限检查方法

**角色权限映射**:

| 角色 | 权限 |
|------|------|
| ADMIN | 全部权限 + 用户管理 + 系统日志 |
| USER | 数据查看、交易执行、账户管理、策略管理 |

**代码量减少**: 约90行 (45%)
