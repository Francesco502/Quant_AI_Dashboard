## 🚀 环境搭建

### 1. 系统要求

- **操作系统**: Windows 10+, Linux, macOS
- **Python**: 3.10 或更高版本（推荐 3.11+）
- **内存**: 至少 4GB RAM（推荐 8GB+）
- **磁盘**: 至少 2GB 可用空间

### 2. 安装Python

#### Windows
```bash
# 下载并安装 Python 3.11+
# https://www.python.org/downloads/

# 验证安装
python --version
pip --version
```

#### Linux/macOS
```bash
# 使用包管理器安装
# Ubuntu/Debian
sudo apt update
sudo apt install python3.11 python3-pip

# macOS (使用 Homebrew)
brew install python@3.11

# 验证安装
python3 --version
pip3 --version
```

### 3. 克隆项目

```bash
git clone <repository-url>
cd Quant_AI_Dashboard
```

### 4. 创建虚拟环境

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate
```

### 5. 安装依赖

```bash
# 升级pip
pip install --upgrade pip

# 安装项目依赖
pip install -r requirements.txt

# 安装开发依赖（可选）
pip install pytest pytest-cov pytest-mock black flake8 mypy
```

### 6. 配置环境变量

```bash
# 复制环境变量模板
cp env.example .env

# 编辑 .env 文件，设置必要的配置
# - APP_LOGIN_PASSWORD: Dashboard登录密码
# - ALPHA_VANTAGE_KEY: Alpha Vantage API密钥（可选）
# - TUSHARE_TOKEN: Tushare Token（可选）
```

### 7. 验证安装

```bash
# 运行测试
pytest tests/ -v

# 启动Dashboard
streamlit run app.py
```

## 📝 代码规范

### 1. Python代码风格

项目遵循 **PEP 8** 代码规范。

#### 使用工具检查

```bash
# 使用flake8检查
flake8 core/ api/ --max-line-length=120 --ignore=E501,W503

# 使用black格式化（可选）
black core/ api/ --line-length=120

# 使用mypy类型检查（可选）
mypy core/ api/ --ignore-missing-imports
```

#### 命名规范

- **模块名**: 小写字母，单词间用下划线 (`data_service.py`)
- **类名**: 驼峰命名 (`DataService`, `RiskMonitor`)
- **函数名**: 小写字母，单词间用下划线 (`load_price_data`)
- **常量名**: 全大写，单词间用下划线 (`MAX_POSITION_SIZE`)
- **变量名**: 小写字母，单词间用下划线 (`current_price`)

#### 代码示例

```python
# ✅ 好的代码风格
class DataService:
    """数据服务类"""
    
    MAX_RETRIES = 3
    
    def load_price_data(self, tickers: List[str], days: int) -> pd.DataFrame:
        """加载价格数据
        
        Args:
            tickers: 标的代码列表
            days: 天数
            
        Returns:
            价格数据DataFrame
        """
        # 实现逻辑
        pass

# ❌ 不好的代码风格
class dataservice:  # 类名应该用驼峰
    max_retries = 3  # 常量应该全大写
    
    def LoadPriceData(self, tickers, days):  # 函数名应该小写
        # 缺少文档字符串
        pass
```

### 2. 文档字符串

所有公共函数、类、模块都应该有文档字符串。

```python
def calculate_var(
    returns: pd.Series,
    confidence_level: float = 0.05,
    method: str = "historical"
) -> float:
    """计算风险价值 (VaR)
    
    Args:
        returns: 收益率序列
        confidence_level: 置信水平，默认0.05（95%置信度）
        method: 计算方法，可选 "historical" 或 "parametric"
        
    Returns:
        VaR值（负数表示损失）
        
    Raises:
        ValueError: 当confidence_level不在(0, 1)范围内时
        
    Example:
        >>> returns = pd.Series([0.01, -0.02, 0.03, -0.01])
        >>> var = calculate_var(returns, confidence_level=0.05)
        >>> print(f"VaR: {var:.4f}")
    """
    pass
```

### 3. 类型注解

尽量使用类型注解，提高代码可读性。

```python
from typing import List, Dict, Optional, Tuple

def process_signals(
    signals: List[Dict[str, float]],
    threshold: float = 0.5
) -> Tuple[List[str], List[float]]:
    """处理信号列表"""
    pass
```

### 4. 错误处理

```python
# ✅ 好的错误处理
try:
    data = load_price_data(tickers, days)
except ValueError as e:
    logger.error(f"数据加载失败: {e}")
    raise
except Exception as e:
    logger.exception("未知错误")
    raise RuntimeError(f"数据加载失败: {e}") from e

# ❌ 不好的错误处理
try:
    data = load_price_data(tickers, days)
except:  # 不要捕获所有异常
    pass  # 不要忽略错误
```

```python
import logging

logger = logging.getLogger(__name__)

```

## 🔄 开发流程

### 1. 创建功能分支

```bash
# 从main分支创建新分支
git checkout -b feature/your-feature-name

# 或从main分支创建bug修复分支
git checkout -b fix/bug-description
```

### 2. 开发新功能

1. **编写代码**
   - 遵循代码规范
   - 添加文档字符串
   - 使用类型注解

2. **编写测试**
   - 为新功能编写单元测试
   - 确保测试覆盖主要场景
   - 运行测试确保通过

3. **更新文档**
   - 更新相关文档
   - 添加使用示例
   - 更新API文档

### 3. 提交代码

```bash
# 查看修改
git status
git diff

# 添加修改
git add .

# 提交（遵循提交规范）
git commit -m "feat: 添加新功能描述"

# 推送到远程
git push origin feature/your-feature-name
```

### 4. 创建Pull Request

1. 在GitHub/GitLab上创建PR
2. 填写PR描述，说明：
   - 功能说明
   - 修改内容
   - 测试情况
   - 相关Issue
3. 等待代码审查
4. 根据反馈修改
5. 合并到main分支

## 📋 提交规范

项目使用 **Conventional Commits** 规范。

### 提交格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型 (type)

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式（不影响功能）
- `refactor`: 重构代码
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具相关

### 示例

```bash
# 新功能
git commit -m "feat(risk): 添加实时风险监控功能"

# 修复bug
git commit -m "fix(data): 修复数据加载失败的问题"

# 文档更新
git commit -m "docs: 更新API文档"

# 重构
git commit -m "refactor(trading): 重构交易引擎代码"

# 性能优化
git commit -m "perf(cache): 优化缓存策略，提升查询性能"
```

## 🧪 测试指南

### 1. 运行测试

```bash
# 运行所有测试
pytest tests/ -v

# 运行特定类型测试
pytest tests/unit/ -v              # 单元测试
pytest tests/integration/ -v       # 集成测试
pytest tests/performance/ -v       # 性能测试

# 运行特定测试文件
pytest tests/unit/test_risk_analysis.py -v

# 运行特定测试方法
pytest tests/unit/test_risk_analysis.py::TestRiskAnalysis::test_calculate_var -v
```

### 2. 生成覆盖率报告

```bash
# 生成HTML覆盖率报告
pytest tests/ --cov=core --cov=api --cov-report=html

# 查看报告
# 打开 htmlcov/index.html
```

### 3. 编写测试

```python
import pytest
from core.risk_analysis import calculate_var

class TestRiskAnalysis:
    """风险分析测试类"""
    
    def test_calculate_var_positive_returns(self):
        """测试正收益率的VaR计算"""
        returns = pd.Series([0.01, 0.02, 0.03])
        var = calculate_var(returns, confidence_level=0.05)
        assert var < 0  # VaR应该是负数
    
    def test_calculate_var_negative_returns(self):
        """测试负收益率的VaR计算"""
        returns = pd.Series([-0.01, -0.02, -0.03])
        var = calculate_var(returns, confidence_level=0.05)
        assert var < 0
    
    @pytest.mark.parametrize("confidence_level", [0.01, 0.05, 0.1])
    def test_calculate_var_different_confidence(self, confidence_level):
        """测试不同置信水平的VaR计算"""
        returns = pd.Series([0.01, -0.02, 0.03, -0.01, 0.02])
        var = calculate_var(returns, confidence_level=confidence_level)
        assert var < 0
```

## 🐛 调试技巧

### 1. 使用日志

```python

# 设置日志级别
logging.basicConfig(
    level=logging.DEBUG,
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s'
)

logger = logging.getLogger(__name__)

# 在代码中添加日志
logger.debug("调试信息")
logger.info("执行信息")
```

### 2. 使用断点调试

```python
# 使用pdb调试
import pdb

def some_function():
    pdb.set_trace()  # 设置断点
    # 代码执行到这里会暂停
    pass
```

### 3. 使用IDE调试

- **VS Code**: 使用Python调试器
- **PyCharm**: 使用内置调试器
- **Jupyter**: 使用交互式调试

### 4. 性能分析

```python
import cProfile
import pstats

# 性能分析
profiler = cProfile.Profile()
profiler.enable()

# 执行代码
your_function()

profiler.disable()
stats = pstats.Stats(profiler)
stats.sort_stats('cumulative')
stats.print_stats(10)  # 打印前10个最耗时的函数
```
