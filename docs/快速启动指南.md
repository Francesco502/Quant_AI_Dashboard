## ✅ 前提条件

在开始之前，请确保您的设备已安装以下软件：

1.  **Python**: 3.10 或更高版本 (推荐 3.11)
2.  **Node.js**: 18.0 或更高版本 (推荐 20.x LTS)
3.  **Git**: 用于克隆代码库

## 🚀 第一步：环境准备

### 1. 克隆代码 (如果您还没有)

```bash
git clone <repository-url>
cd Quant_AI_Dashboard-main
```

### 2. 后端依赖安装

建议使用虚拟环境以避免污染全局 Python 环境。

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# Linux/macOS
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install --upgrade pip
pip install -r requirements.txt
```

### 3. 前端依赖安装

```bash
cd web
npm install
# 或者使用 yarn / pnpm
# npm install -g pnpm
# pnpm install
cd ..
```

### 4. 配置文件设置

**后端配置**:
复制 `env.example` 到 `.env` 并按需修改。
```bash
cp env.example .env
```
*主要配置项*:
- `APP_LOGIN_PASSWORD`: Dashboard 登录密码 (默认: admin)
- `TUSHARE_TOKEN`: (可选) Tushare 数据源 Token

**前端配置**:
复制 `web/.env.example` 到 `web/.env.local`。
```bash
cd web
cp .env.example .env.local
cd ..
```

## 🖥️ 第二步：启动后端服务 (Backend)

后端服务基于 FastAPI，提供 RESTful API 接口。

打开一个新的终端窗口（**终端 1**）：

```bash
# 确保已激活虚拟环境
# venv\Scripts\activate (Windows)

python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

看到 `Uvicorn running on http://0.0.0.0:8000` 即表示启动成功。

## 🌐 第三步：启动前端界面 (Frontend)

前端基于 Next.js，提供用户交互界面。

打开一个新的终端窗口（**终端 2**）：

```bash
cd web
npm run dev
```

看到 `Ready in ...` 和 `http://localhost:8686` 即表示启动成功。

## 🤖 第四步：启动后台守护进程 (Daemon)

后台守护进程负责执行定时任务，如：
- 每日收盘后更新市场数据
- 执行全市场扫描策略
- 自动运行模拟交易

打开一个新的终端窗口（**终端 3**）：

```bash
# 确保已激活虚拟环境
python run_daemon.py
```

看到 `daemon: 开始进入调度循环 ...` 即表示启动成功。

## ✅ 验证运行

1.  **访问 Dashboard**: 打开浏览器访问 [http://localhost:8686](http://localhost:8686)。
2.  **API 文档**: 访问 [http://localhost:8000/docs](http://localhost:8000/docs) 查看后端接口。

## ❓ 常见问题

**Q: 启动后端时提示 `ModuleNotFoundError`?**
A: 请确认是否激活了虚拟环境，并且已成功运行 `pip install -r requirements.txt`。

**Q: 前端页面显示 "Network Error" 或无法连接?**
A: 请检查后端服务（端口 8000）是否正常运行。确保前端 `.env.local` 中的 `NEXT_PUBLIC_API_URL` 配置正确（通常为 `http://localhost:8000`）。

**Q: 守护进程启动失败?**
A: 检查 `daemon_config.json` 是否生成，如果报错请查看 `logs/daemon.log` 获取详细错误信息。确保没有其他 Python 进程占用了文件锁。
