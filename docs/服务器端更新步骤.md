# 服务器端部署与更新步骤

> 本文档提供在个人服务器上通过 Docker 部署和更新 Quant-AI Dashboard 的完整操作指南。
>
> **架构**: Next.js 前端 + FastAPI 后端 + Nginx 反向代理  
> **更新日期**: 2026-02-11

---

## 一、架构概览

```
用户浏览器
    │
    ▼
┌──────────┐
│  Nginx   │ :80 / :443（统一入口）
│  反向代理 │
└────┬─────┘
     │
     ├── /         → frontend:8686  (Next.js)
     ├── /api/*    → backend:8685   (FastAPI)
     └── /ws/*     → backend:8685   (WebSocket)

┌──────────┐
│  daemon  │  后台守护进程（数据更新、定时任务）
└──────────┘
```

**服务列表**:
| 服务 | 端口 | 说明 |
|------|------|------|
| nginx | 80 (443) | 反向代理，统一入口 |
| backend | 8685 | FastAPI API 服务 |
| frontend | 8686 | Next.js 前端 |
| daemon | - | 后台守护进程 |

## 二、首次部署

### 前置条件

- Linux 服务器（推荐 Ubuntu 20.04+）
- Docker 20.10+ 和 Docker Compose v2+
- Git

### 步骤

```bash
# 1. 克隆项目
git clone https://github.com/your-repo/Quant_AI_Dashboard.git
cd Quant_AI_Dashboard

# 2. 配置环境变量
cp env.example .env
# 编辑 .env，设置密码和 API 密钥
nano .env  # 或 vim .env

# 3. 创建必要的目录
mkdir -p data logs nginx strategies models

# 4. 构建并启动所有服务
docker compose build --no-cache
docker compose up -d

# 5. 检查服务状态
docker compose ps

# 6. 查看日志确认启动正常
docker compose logs -f
```

### 验证部署成功

```bash
# 检查所有容器运行状态
docker compose ps
# 应看到 4 个容器都是 running 状态

# 检查后端 API 健康
curl http://localhost:8685/api/health
# 应返回 {"status":"healthy","service":"quant-ai-api"}

# 检查前端
curl -I http://localhost:8686
# 应返回 HTTP 200 (前端端口 8686)

# 检查 Nginx 统一入口
curl -I http://localhost
# 应返回 HTTP 200

# 浏览器访问
# http://your-server-ip
```

## 三、日常更新

### 快速更新（推荐）

#### Linux/macOS 服务器

```bash
# 1. 进入项目目录
cd /path/to/Quant_AI_Dashboard

# 2. 备份数据（重要！）
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz data/ logs/
cp .env .env.backup

# 3. 拉取最新代码
git pull origin main

# 4. 停止当前服务
docker compose down

# 5. 重新构建（代码有变更时）
docker compose build --no-cache

# 6. 启动新版本
docker compose up -d

# 7. 检查状态
docker compose ps
docker compose logs --tail=50
```

#### Windows 服务器

```powershell
# 1. 进入项目目录
cd D:\path\to\Quant_AI_Dashboard

# 2. 备份数据
$backupDir = "backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
New-Item -ItemType Directory -Path $backupDir -Force
Compress-Archive -Path data\*, logs\* -DestinationPath "$backupDir\data.zip"
Copy-Item .env "$backupDir\"

# 3. 拉取最新代码
git pull origin main

# 4. 停止 -> 构建 -> 启动
docker compose down
docker compose build --no-cache
docker compose up -d

# 5. 检查状态
docker compose ps
docker compose logs --tail=50
```

## 四、一键更新脚本

### Linux/macOS

创建 `update_server.sh`:

```bash
#!/bin/bash
set -e

echo "=========================================="
echo "  Quant-AI Dashboard 服务器更新脚本"
echo "=========================================="
echo ""

# 检查是否在项目目录
if [ ! -f "docker-compose.yml" ]; then
    echo "[错误] 请在项目根目录运行此脚本"
    exit 1
fi

# 1. 备份
echo "[步骤1] 备份数据..."
BACKUP_DIR="backup_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"
[ -d "data" ] && tar -czf "$BACKUP_DIR/data.tar.gz" data/ 2>/dev/null || true
[ -f ".env" ] && cp .env "$BACKUP_DIR/"
echo "  备份完成: $BACKUP_DIR"

# 2. 拉取代码
echo "[步骤2] 拉取最新代码..."
git pull origin main

# 3. 停止服务
echo "[步骤3] 停止当前服务..."
docker compose down

# 4. 构建镜像
echo "[步骤4] 构建镜像..."
docker compose build --no-cache

# 5. 启动服务
echo "[步骤5] 启动服务..."
docker compose up -d

# 6. 等待
echo "[等待] 等待服务启动（15秒）..."
sleep 15

# 7. 检查
echo "[步骤6] 检查服务状态..."
docker compose ps
echo ""

# 8. 健康检查
echo "[步骤7] 健康检查..."
if curl -sf http://localhost:8685/api/health > /dev/null 2>&1; then
    echo "  后端 API: 正常"
else
    echo "  后端 API: 异常！请检查日志"
fi

if curl -sf http://localhost > /dev/null 2>&1; then
    echo "  Nginx 入口: 正常"
else
    echo "  Nginx 入口: 异常！请检查日志"
fi

echo ""
echo "=========================================="
echo "  更新完成！访问: http://your-server-ip"
echo "=========================================="
echo ""
echo "查看日志: docker compose logs -f"
echo "备份位置: $BACKUP_DIR"
```

```bash
chmod +x update_server.sh
./update_server.sh
```

### Windows

创建 `update_server.ps1`:

```powershell
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host "  Quant-AI Dashboard 服务器更新脚本" -ForegroundColor Cyan
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host ""

    Write-Host "[错误] 请在项目根目录运行此脚本" -ForegroundColor Red

if (Test-Path "data") {
    Compress-Archive -Path data\* -DestinationPath "$backupDir\data.zip" -Force
}
if (Test-Path ".env") { Copy-Item .env "$backupDir\" }
Write-Host "  备份完成: $backupDir" -ForegroundColor Green

# 2. 拉取代码
Write-Host "[步骤2] 拉取最新代码..." -ForegroundColor Yellow
git pull origin main

# 3. 停止 -> 构建 -> 启动
Write-Host "[步骤3] 停止服务..." -ForegroundColor Yellow
docker compose down

Write-Host "[步骤4] 构建镜像..." -ForegroundColor Yellow
docker compose build --no-cache

Write-Host "[步骤5] 启动服务..." -ForegroundColor Yellow
docker compose up -d

Write-Host "[等待] 等待服务启动（15秒）..." -ForegroundColor Yellow
Start-Sleep -Seconds 15

Write-Host "[步骤6] 检查状态..." -ForegroundColor Yellow
docker compose ps

Write-Host ""
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host "  更新完成！" -ForegroundColor Green
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host "  查看日志: docker compose logs -f" -ForegroundColor White
Write-Host "  备份位置: $backupDir" -ForegroundColor White
```

```powershell
powershell -ExecutionPolicy Bypass -File update_server.ps1
```

## 六、HTTPS 配置（推荐）

### 使用 Let's Encrypt 免费证书

```bash
# 1. 安装 certbot
sudo apt install certbot

# 2. 获取证书（先临时停止 nginx）
docker compose stop nginx
sudo certbot certonly --standalone -d your-domain.com

# 3. 复制证书到项目目录
mkdir -p nginx/certs
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem nginx/certs/
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem nginx/certs/
sudo chown $USER:$USER nginx/certs/*.pem

# 4. 修改 docker-compose.yml，取消 443 端口注释
# 5. 修改 nginx/nginx.conf，取消 HTTPS server 块注释，替换 your-domain.com
# 6. 重启 nginx
docker compose up -d nginx
```

## 八、故障排查

### 问题1: 前端无法访问后端 API

```bash
# 检查后端是否正常
curl http://localhost:8685/api/health

# 检查容器网络
docker network inspect quant_ai_dashboard-main_quant-network

# 检查 Nginx 配置
docker exec quant-nginx nginx -t
```

### 问题2: 容器启动失败

```bash
# 查看详细日志
docker compose logs backend
docker compose logs frontend

# 检查端口占用
netstat -tulpn | grep -E '80|8686|8685'
# 或
ss -tulpn | grep -E '80|8686|8685'
```

### 问题3: 构建失败

```bash
# 清理 Docker 构建缓存
docker builder prune -f

# 单独构建某个服务排查问题
docker compose build backend --no-cache --progress=plain
docker compose build frontend --no-cache --progress=plain
```

## 九、注意事项

1. **数据安全**: 更新前务必备份 `data/` 目录和 `.env` 文件
2. **内存需求**: 后端（含 AI 模型）建议至少 4GB RAM，前端构建需要约 1GB
3. **磁盘空间**: Docker 镜像约占 3-5GB，建议预留 10GB+ 空间
4. **网络**: 首次构建需要下载依赖，确保服务器可访问 PyPI 和 npm 仓库
5. **安全**: 生产环境务必修改 `.env` 中的 `SECRET_KEY`，设置强密码
