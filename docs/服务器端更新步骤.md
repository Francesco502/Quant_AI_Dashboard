# 服务器端更新步骤

> 本文档提供服务器端从 v0.1.3 更新到 v0.1.4 的详细操作步骤

**目标版本**: v0.1.4  
**更新日期**: 2025-01-XX

---

## 📋 快速更新（推荐）

### Linux/macOS 服务器

```bash
# 1. 进入项目目录
cd /path/to/Quant_AI_Dashboard

# 2. 备份数据（重要！）
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz data/
cp daemon_config.json daemon_config.json.backup

# 3. 停止当前服务
docker compose down

# 4. 拉取新镜像
docker pull francescoli/quant_ai_dashboard:v0.1.4

# 5. 更新docker-compose.yml（如果版本不是v0.1.4）
sed -i 's/v0\.1\.3/v0.1.4/g' docker-compose.yml

# 6. 启动新版本
docker compose up -d

# 7. 检查状态
docker compose ps
docker compose logs -f dashboard
```

### Windows 服务器

```powershell
# 1. 进入项目目录
cd D:\path\to\Quant_AI_Dashboard

# 2. 备份数据
$backupDir = "backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
New-Item -ItemType Directory -Path $backupDir
Compress-Archive -Path data\* -DestinationPath "$backupDir\data.zip"
Copy-Item daemon_config.json "$backupDir\"

# 3. 停止当前服务
docker compose down

# 4. 拉取新镜像
docker pull francescoli/quant_ai_dashboard:v0.1.4

# 5. 更新docker-compose.yml（如果版本不是v0.1.4）
(Get-Content docker-compose.yml) -replace 'v0\.1\.3', 'v0.1.4' | Set-Content docker-compose.yml

# 6. 启动新版本
docker compose up -d

# 7. 检查状态
docker compose ps
docker compose logs -f dashboard
```

## 📝 详细步骤说明

### 步骤1: 备份数据

**为什么重要**: 防止更新过程中数据丢失

```bash
# 备份数据目录
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz data/

# 备份配置文件
cp daemon_config.json daemon_config.json.backup
cp risk_config.json risk_config.json.backup 2>/dev/null || true

# 备份数据库（如果使用）
cp data/quant.db data/quant.db.backup 2>/dev/null || true
```

### 步骤2: 停止当前服务

```bash
# 停止并删除容器
docker compose down

# 确认容器已停止
docker compose ps
# 应该显示空列表或只有已停止的容器
```

### 步骤3: 检查docker-compose.yml

确保镜像版本为 `v0.1.4`：

```bash
# 查看当前配置
grep "image:" docker-compose.yml

# 如果显示 v0.1.3，需要更新
sed -i 's/v0\.1\.3/v0.1.4/g' docker-compose.yml

# 验证更新
grep "image:" docker-compose.yml
# 应该显示: image: francescoli/quant_ai_dashboard:v0.1.4
```

### 步骤4: 拉取新镜像

```bash
# 拉取v0.1.4镜像
docker pull francescoli/quant_ai_dashboard:v0.1.4

# 验证镜像已下载
docker images | grep quant_ai_dashboard
# 应该看到 v0.1.4 镜像
```

### 步骤5: 启动新版本

```bash
# 启动服务
docker compose up -d

# 查看启动状态
docker compose ps
# 应该看到两个容器都是 "Up" 状态
```

### 步骤6: 验证更新

```bash
# 方法1: 检查容器使用的镜像
docker inspect quant-dashboard | grep -A 5 "Image"
# 应该显示: francescoli/quant_ai_dashboard:v0.1.4

# 方法2: 在容器内检查版本
docker exec quant-dashboard python -c "from core.version import get_version; print(get_version())"
# 应该输出: 0.1.4

# 方法3: 访问Dashboard查看
# 打开浏览器访问 http://your-server:8501
# 查看页面底部或关于页面的版本信息
```

### 步骤7: 检查日志

```bash
# 查看Dashboard日志
docker compose logs -f dashboard

# 查看Daemon日志
docker compose logs -f daemon

# 检查是否有错误
docker compose logs dashboard | grep -i error
docker compose logs daemon | grep -i error
```

## 🔄 一键更新脚本

### Linux/macOS 服务器

创建 `update_server.sh`:

```bash
#!/bin/bash
# 服务器端更新脚本

set -e

echo "=========================================="
echo "服务器端更新脚本 - v0.1.3 -> v0.1.4"
echo "=========================================="
echo ""

# 检查是否在项目目录
if [ ! -f "docker-compose.yml" ]; then
    echo "❌ 错误: 请在项目根目录运行此脚本"
    exit 1
fi

# 1. 备份数据
echo "📦 步骤1: 备份数据..."
BACKUP_DIR="backup_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

if [ -d "data" ]; then
    tar -czf "$BACKUP_DIR/data.tar.gz" data/ 2>/dev/null || echo "  警告: 数据目录备份失败"
fi

if [ -f "daemon_config.json" ]; then
    cp daemon_config.json "$BACKUP_DIR/"
fi

echo "✅ 备份完成: $BACKUP_DIR"
echo ""

# 2. 停止服务
echo "🛑 步骤2: 停止当前服务..."
docker compose down
echo "✅ 服务已停止"
echo ""

# 3. 更新配置
echo "📝 步骤3: 更新docker-compose.yml..."
if grep -q "v0\.1\.3" docker-compose.yml; then
    sed -i.bak 's/v0\.1\.3/v0.1.4/g' docker-compose.yml
    echo "✅ 配置已更新"
else
    echo "✅ 配置已是最新版本"
fi
echo ""

# 4. 拉取镜像
echo "⬇️  步骤4: 拉取新镜像..."
docker pull francescoli/quant_ai_dashboard:v0.1.4
echo "✅ 镜像拉取完成"
echo ""

# 5. 启动服务
echo "🚀 步骤5: 启动新版本..."
docker compose up -d
echo "✅ 服务启动中..."
echo ""

# 6. 等待启动
echo "⏳ 等待服务启动（10秒）..."
sleep 10

# 7. 检查状态
echo "🔍 步骤6: 检查服务状态..."
docker compose ps

# 8. 验证版本
echo ""
echo "🔍 步骤7: 验证版本..."
VERSION=$(docker exec quant-dashboard python -c "from core.version import get_version; print(get_version())" 2>/dev/null || echo "unknown")
echo "当前版本: $VERSION"

if [ "$VERSION" = "0.1.4" ]; then
    echo "✅ 版本验证通过"
else
    echo "⚠️  版本验证失败，请检查日志"
fi

echo ""
echo "=========================================="
echo "✅ 更新完成！"
echo "=========================================="
echo ""
echo "📊 查看日志:"
echo "  docker compose logs -f dashboard"
echo "  docker compose logs -f daemon"
echo ""
echo "🌐 访问Dashboard:"
echo "  http://your-server:8501"
echo ""
echo "📦 备份位置: $BACKUP_DIR"
echo ""
```

赋予执行权限并运行：

```bash
chmod +x update_server.sh
./update_server.sh
```

### Windows 服务器

创建 `update_server.ps1`:

```powershell
# 服务器端更新脚本

Write-Host "==========================================" -ForegroundColor Cyan
Write-Host "服务器端更新脚本 - v0.1.3 -> v0.1.4" -ForegroundColor Cyan
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host ""

# 检查是否在项目目录
if (-not (Test-Path "docker-compose.yml")) {
    Write-Host "[错误] 请在项目根目录运行此脚本" -ForegroundColor Red
    exit 1
}

# 1. 备份数据
Write-Host "[步骤1] 备份数据..." -ForegroundColor Yellow
$backupDir = "backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
New-Item -ItemType Directory -Path $backupDir -Force | Out-Null

if (Test-Path "data") {
    Compress-Archive -Path data\* -DestinationPath "$backupDir\data.zip" -Force
}

if (Test-Path "daemon_config.json") {
    Copy-Item daemon_config.json "$backupDir\"
}

Write-Host "[完成] 备份完成: $backupDir" -ForegroundColor Green
Write-Host ""

# 2. 停止服务
Write-Host "[步骤2] 停止当前服务..." -ForegroundColor Yellow
docker compose down
Write-Host "[完成] 服务已停止" -ForegroundColor Green
Write-Host ""

# 3. 更新配置
Write-Host "[步骤3] 更新docker-compose.yml..." -ForegroundColor Yellow
$content = Get-Content docker-compose.yml -Raw
if ($content -match "v0\.1\.3") {
    $content = $content -replace 'v0\.1\.3', 'v0.1.4'
    $content | Set-Content docker-compose.yml
    Write-Host "[完成] 配置已更新" -ForegroundColor Green
} else {
    Write-Host "[完成] 配置已是最新版本" -ForegroundColor Green
}
Write-Host ""

# 4. 拉取镜像
Write-Host "[步骤4] 拉取新镜像..." -ForegroundColor Yellow
docker pull francescoli/quant_ai_dashboard:v0.1.4
Write-Host "[完成] 镜像拉取完成" -ForegroundColor Green
Write-Host ""

# 5. 启动服务
Write-Host "[步骤5] 启动新版本..." -ForegroundColor Yellow
docker compose up -d
Write-Host "[完成] 服务启动中..." -ForegroundColor Green
Write-Host ""

# 6. 等待启动
Write-Host "[等待] 等待服务启动（10秒）..." -ForegroundColor Yellow
Start-Sleep -Seconds 10

# 7. 检查状态
Write-Host "[步骤6] 检查服务状态..." -ForegroundColor Yellow
docker compose ps
Write-Host ""

# 8. 验证版本
Write-Host "[步骤7] 验证版本..." -ForegroundColor Yellow
try {
    $version = docker exec quant-dashboard python -c "from core.version import get_version; print(get_version())" 2>&1
    Write-Host "当前版本: $version" -ForegroundColor White
    if ($version -eq "0.1.4") {
        Write-Host "[完成] 版本验证通过" -ForegroundColor Green
    } else {
        Write-Host "[警告] 版本验证失败，请检查日志" -ForegroundColor Yellow
    }
} catch {
    Write-Host "[警告] 无法验证版本，请手动检查" -ForegroundColor Yellow
}

Write-Host ""
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host "[完成] 更新完成！" -ForegroundColor Green
Write-Host "==========================================" -ForegroundColor Cyan
Write-Host ""
Write-Host "[查看日志]" -ForegroundColor Yellow
Write-Host "  docker compose logs -f dashboard" -ForegroundColor White
Write-Host "  docker compose logs -f daemon" -ForegroundColor White
Write-Host ""
Write-Host "[访问Dashboard]" -ForegroundColor Yellow
Write-Host "  http://your-server:8501" -ForegroundColor White
Write-Host ""
Write-Host "[备份位置] $backupDir" -ForegroundColor Yellow
Write-Host ""
```

运行：

```powershell
powershell -ExecutionPolicy Bypass -File update_server.ps1
```

## ⚠️ 注意事项

### 1. 更新时机

- **推荐时间**: 非业务高峰期（如凌晨或周末）
- **预计时间**: 5-10分钟（取决于网络速度）
- **停机时间**: 约1-2分钟（停止和启动容器）

### 2. 数据安全

- ✅ **必须备份**: 更新前务必备份 `data/` 目录
- ✅ **配置文件**: 备份 `daemon_config.json` 和 `risk_config.json`
- ✅ **数据库**: 如果使用SQLite，备份 `data/quant.db`

### 3. 网络要求

- 确保服务器可以访问 Docker Hub
- 如果无法访问，使用镜像导出/导入方式（见下方）

### 4. 回滚方案

如果更新后出现问题，可以快速回滚：

```bash
# 停止新版本
docker compose down

# 恢复配置
sed -i 's/v0\.1\.4/v0.1.3/g' docker-compose.yml

# 启动旧版本（如果本地还有旧镜像）
docker compose up -d

# 或拉取旧镜像
docker pull francescoli/quant_ai_dashboard:v0.1.3
docker compose up -d
```

## 🔧 无法访问Docker Hub的情况

如果服务器无法访问 Docker Hub，可以使用镜像导出/导入方式：

### 在本地机器上导出镜像

```bash
# 导出镜像
docker save francescoli/quant_ai_dashboard:v0.1.4 \
  -o quant_ai_dashboard_v0.1.4.tar

# 压缩（可选，减少传输大小）
gzip quant_ai_dashboard_v0.1.4.tar
```

### 传输到服务器

```bash
# 使用scp传输
scp quant_ai_dashboard_v0.1.4.tar.gz user@server:/path/to/Quant_AI_Dashboard/

# 或使用其他方式（FTP、U盘等）
```

### 在服务器上导入

```bash
# 解压（如果压缩了）
gunzip quant_ai_dashboard_v0.1.4.tar.gz

# 导入镜像
docker load -i quant_ai_dashboard_v0.1.4.tar

# 验证导入
docker images | grep quant_ai_dashboard

# 然后按照正常步骤更新docker-compose.yml并启动
```

## 🆘 故障排查

### 问题1: 拉取镜像失败

```bash
# 检查网络连接
ping registry-1.docker.io

# 检查Docker Hub访问
curl -I https://hub.docker.com

# 使用镜像加速器（如果在中国）
# 配置 /etc/docker/daemon.json
```

### 问题2: 容器启动失败

```bash
# 查看详细日志
docker compose logs dashboard
docker compose logs daemon

# 检查端口占用
netstat -tulpn | grep 8501

# 检查数据目录权限
ls -la data/
```

### 问题3: 版本号不正确

```bash
# 强制重新拉取
docker pull francescoli/quant_ai_dashboard:v0.1.4

# 强制重新创建容器
docker compose up -d --force-recreate
```
