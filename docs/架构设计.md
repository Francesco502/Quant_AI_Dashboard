## 🎯 系统概述

Quant-AI-Dashboard 是一个分层架构的量化交易系统，采用模块化设计，支持高并发、可扩展、易维护。

### 系统特点

- **分层架构**：清晰的层次划分，职责明确
- **模块化设计**：高内聚、低耦合
- **可扩展性**：易于添加新功能模块
- **数据驱动**：统一的数据接口和存储
- **生产级**：完整的监控、告警、日志系统

## 🔧 核心模块

### 1. 数据服务层 (`core/data_service.py`)

**职责**：
- 多数据源管理（AkShare、Tushare、yfinance）
- 数据获取和缓存
- 数据质量验证
- 数据修复

**关键接口**：
```python
def load_price_data(tickers, days, data_sources)
def load_ohlcv_data(ticker, start_date, end_date)
def validate_data(data, ticker)
def repair_data(data, issues)
```

### 2. AI预测模块 (`core/advanced_forecasting.py`)

**职责**：
- 模型训练和评估
- 模型注册和管理
- 预测生成
- 模型版本控制

**支持的模型**：
- XGBoost
- LightGBM
- Random Forest
- LSTM/GRU
- Prophet

### 3. 策略引擎 (`core/strategy_engine.py`)

**职责**：
- 策略定义和执行
- 信号生成
- 策略评估
- 策略管理

**策略类型**：
- 技术指标策略
- AI驱动策略
- 混合策略（Ensemble）

### 4. 交易引擎 (`core/trading_engine.py`)

**职责**：
- 订单生成
- 仓位管理
- 交易执行
- 账户管理

**关键功能**：
- 等权重调仓
- 风险检查集成
- 滑点模型
- 执行算法（TWAP/VWAP）

### 5. 风险管理 (`core/risk_monitor.py`)

**职责**：
- 实时风险监控
- 仓位限制检查
- 止损止盈管理
- 风险事件告警

**风险指标**：
- VaR/CVaR
- 最大回撤
- 集中度风险
- 损失限制

### 6. 订单管理 (`core/order_manager.py`)

**职责**：
- 订单生命周期管理
- 订单状态追踪
- 成交记录管理
- 订单历史查询

### 7. 系统监控 (`core/monitoring/`)

**职责**：
- 系统指标收集
- 健康检查
- 告警管理
- 性能监控

### 8. 全市场扫描 (`core/market_scanner.py`)

**职责**：
- 获取全市场（A股/港股/美股）标的列表
- 分批次执行选股策略
- 整合技术指标与形态识别
- 生成候选股票池

### 9. 回测系统 (`core/backtest.py`)

**职责**：
- 历史数据回放
- 策略逻辑模拟执行
- 权益曲线与绩效指标计算 (Sharpe, Drawdown)
- 可视化结果输出

### 10. 后台守护进程 (`core/daemon.py`)

**职责**：
- 任务调度 (Scheduler)
- 定时数据更新
- 自动化交易执行
- 离线模型训练

```

    ↓

    ↓

```

    ↓

    ↓
模型评估 (ModelEvaluator)
    ↓
模型注册 (ModelRegistry)
    ↓
模型部署 (models/*.pkl)
    ↓
在线预测 (advanced_forecasting.py)
```

```

## 🎨 设计原则

### 1. 单一职责原则

每个模块只负责一个明确的功能，例如：
- `data_service.py` 只负责数据获取
- `risk_monitor.py` 只负责风险监控
- `order_manager.py` 只负责订单管理

### 2. 依赖倒置原则

高层模块不依赖低层模块，都依赖抽象接口：
- 数据服务使用统一接口，不依赖具体数据源
- 策略引擎使用策略接口，不依赖具体策略实现

### 3. 开闭原则

对扩展开放，对修改关闭：
- 新增数据源只需实现数据源接口
- 新增策略只需继承BaseStrategy
- 新增模型只需实现Forecaster接口

### 4. 接口隔离原则

使用多个专门的接口，而不是单一的总接口：
- 数据服务接口、预测接口、交易接口分离
- 避免接口臃肿，提高灵活性

### 5. 模块化设计

- 高内聚：相关功能集中在同一模块
- 低耦合：模块间依赖最小化
- 可测试：每个模块可独立测试

## 🔄 扩展性设计

### 添加新数据源

1. 实现数据源接口
2. 在 `data_service.py` 中注册
3. 更新配置和文档

### 添加新模型

1. 实现 `Forecaster` 接口
2. 在 `ModelManager` 中注册
3. 更新训练流水线

### 添加新策略

1. 继承 `BaseStrategy`
2. 实现策略逻辑
3. 在 `StrategyManager` 中注册
