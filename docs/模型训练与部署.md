## 🚀 离线训练

### 1. 配置后台训练

编辑 `daemon_config.json`:

```json
{
  "universe": ["159755.SZ", "002611", "006810"],
  "training": {
    "enabled": true,
    "time": "16:00",
    "model_type": "xgboost",  // 可选: "xgboost", "lightgbm", "random_forest", "lstm", "gru"
    "auto_promote": true,
    "generate_signals": true,
    "min_train_days": 60,
    "retrain_interval_days": 7,
    "min_improvement_threshold": 0.02
  }
}
```

### 2. 启动后台训练

```bash
cd Quant_AI_Dashboard
python -m core.daemon
```

后台服务会在指定时间自动训练模型。

### 3. 手动训练单个模型

```python
from core.training_pipeline import TrainingPipeline
from core.data_store import load_local_price_history

pipeline = TrainingPipeline()

# 训练 XGBoost 模型
price_series = load_local_price_history("159755.SZ")
model_id = pipeline.train_model("159755.SZ", price_series, model_type="xgboost")

# 训练 LightGBM 模型
model_id = pipeline.train_model("159755.SZ", price_series, model_type="lightgbm")

# 训练 Random Forest 模型
model_id = pipeline.train_model("159755.SZ", price_series, model_type="random_forest")

# 训练 LSTM 模型
model_id = pipeline.train_model("159755.SZ", price_series, model_type="lstm")

# 训练 GRU 模型
model_id = pipeline.train_model("159755.SZ", price_series, model_type="gru")
```

### 4. 批量训练

```python
from core.training_pipeline import TrainingPipeline

pipeline = TrainingPipeline()

# 批量训练多个标的
tickers = ["159755.SZ", "002611", "006810"]
model_type = "xgboost"

for ticker in tickers:
    try:
        price_series = load_local_price_history(ticker)
        model_id = pipeline.train_model(ticker, price_series, model_type=model_type)
        print(f"{ticker} 训练完成: {model_id}")
    except Exception as e:
        print(f"{ticker} 训练失败: {e}")
```

## 📦 在线部署

### 1. 模型注册

训练完成后，模型会自动注册到 `ModelRegistry`:

```python
from core.advanced_forecasting import ModelRegistry

registry = ModelRegistry()

# 获取生产模型ID
model_id = registry.get_production_model("159755.SZ")

# 获取模型信息
model_info = registry.get_model_info(model_id)
print(f"模型类型: {model_info['model_type']}")
print(f"训练日期: {model_info['trained_at']}")
print(f"评估指标: {model_info['metrics']}")
```

### 2. 使用生产模型预测

#### 在Dashboard中使用

1. 打开 "AI 趋势预测" 页面
2. 选择模型类型（XGBoost、LightGBM等）
3. 勾选 "🏭 优先使用生产模型"
4. 系统会自动加载后台训练好的离线模型（如果存在）

#### 在代码中使用

```python
from core.advanced_forecasting import quick_predict

# 使用生产模型预测
predictions = quick_predict(
    ticker="159755.SZ",
    horizon=5,
    model_type="xgboost",
    use_production_model=True
)

print(predictions)
```

### 3. 模型评估和提升

训练完成后，系统会自动评估模型：

```python
from core.training_pipeline import TrainingPipeline

pipeline = TrainingPipeline()

# 评估模型
metrics = pipeline.evaluate_model("159755.SZ", model_id, model_type="xgboost")
print(f"方向准确率: {metrics['direction_accuracy']:.2%}")
print(f"MAE: {metrics['mae']:.4f}")
print(f"RMSE: {metrics['rmse']:.4f}")

# 对比新旧模型
if pipeline.compare_models(old_model_id, new_model_id):
    # 新模型更好，自动提升为生产模型
    pipeline.set_production_model("159755.SZ", new_model_id)
```

## 🔄 完整工作流程

### 离线训练流程

```
daemon.py (training_job)
  ↓
读取配置 model_type
  ↓
TrainingPipeline.run_training_job(model_type=model_type)
  ↓
TrainingPipeline.train_model(model_type=model_type)
  ↓
ModelManager.train_model(model_type=model_type)
  ↓
根据 model_type 调用对应的训练方法:
  - train_xgboost()
  - train_lightgbm()
  - train_random_forest()
  - train_lstm()
  - train_gru()
  ↓
训练模型 → 保存到磁盘 → 注册到 ModelRegistry
  ↓
评估模型 → 对比新旧模型 → 提升为生产模型（如果更好）
```

### 在线部署流程

```
AI预测页面
  ↓
选择模型类型 + 勾选"优先使用生产模型"
  ↓
hybrid_price_forecast(model_type=model_type, prefer_production=True)
  ↓
predict_with_production_model(model_type=model_type)
  ↓
quick_predict(model_type=model_type, use_production_model=True)
  ↓
1. ModelRegistry.get_production_model(ticker)
2. 检查模型类型是否匹配
3. 从缓存/磁盘加载模型
4. model.predict(horizon)
  ↓
返回预测结果
```

## 💡 使用示例

### 示例1: 完整训练和部署流程

```python
from core.training_pipeline import TrainingPipeline
from core.data_store import load_local_price_history
from core.advanced_forecasting import quick_predict, ModelRegistry

# 1. 训练模型
pipeline = TrainingPipeline()
price_series = load_local_price_history("159755.SZ")
model_id = pipeline.train_model("159755.SZ", price_series, model_type="xgboost")

# 2. 评估模型
metrics = pipeline.evaluate_model("159755.SZ", model_id, model_type="xgboost")
print(f"评估结果: {metrics}")

# 3. 提升为生产模型
registry = ModelRegistry()
old_production = registry.get_production_model("159755.SZ")
if old_production:
    if pipeline.compare_models(old_production, model_id):
        pipeline.set_production_model("159755.SZ", model_id)
        print("新模型已提升为生产模型")
else:
    # 没有旧模型，直接设置为生产模型
    pipeline.set_production_model("159755.SZ", model_id)

# 4. 使用生产模型预测
predictions = quick_predict(
    ticker="159755.SZ",
    horizon=5,
    model_type="xgboost",
    use_production_model=True
)
print(f"预测结果: {predictions}")
```

### 示例2: 批量训练多个标的

```python
from core.training_pipeline import TrainingPipeline
from core.data_store import load_local_price_history

pipeline = TrainingPipeline()
tickers = ["159755.SZ", "002611", "006810"]
model_type = "lightgbm"

results = {}
for ticker in tickers:
    try:
        price_series = load_local_price_history(ticker)
        if len(price_series) < 60:
            print(f"{ticker}: 数据不足，跳过")
            continue
        
        model_id = pipeline.train_model(ticker, price_series, model_type=model_type)
        metrics = pipeline.evaluate_model(ticker, model_id, model_type=model_type)
        
        results[ticker] = {
            "model_id": model_id,
            "metrics": metrics,
            "status": "success"
        }
        print(f"{ticker}: 训练成功，方向准确率 {metrics['direction_accuracy']:.2%}")
    except Exception as e:
        results[ticker] = {
            "status": "failed",
            "error": str(e)
        }
        print(f"{ticker}: 训练失败 - {e}")

# 打印总结
print("\n训练总结:")
for ticker, result in results.items():
    if result["status"] == "success":
        print(f"{ticker}: ✅ {result['metrics']['direction_accuracy']:.2%}")
    else:
        print(f"{ticker}: ❌ {result['error']}")
```

## ⚠️ 注意事项

1. **模型类型匹配**: 确保训练和预测使用相同的 `model_type`
2. **特征版本**: 通过 `features_version` 跟踪特征工程变化
3. **训练时间**: 深度学习模型（LSTM/GRU）训练时间长，建议在非交易时间运行
4. **资源消耗**: Random Forest 内存占用大，LSTM/GRU 训练时间长
5. **模型评估**: 新模型需要经过评估才能提升为生产模型
6. **数据质量**: 训练前确保数据完整，无缺失值
