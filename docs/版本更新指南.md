## ⚠️ 更新前准备

### 1. 检查当前版本

在服务器上检查当前运行的版本：

```bash
# 查看当前容器使用的镜像版本
docker compose ps
docker inspect quant-dashboard | grep Image

# 或在Dashboard中查看版本信息
# 访问 http://your-server:8686，查看页面底部的版本号
```

### 2. 备份重要数据

**重要**: 更新前务必备份以下数据：

```bash
# 进入项目目录
cd /path/to/Quant_AI_Dashboard

# 备份数据目录
tar -czf backup_data_$(date +%Y%m%d_%H%M%S).tar.gz data/

# 备份配置文件
cp daemon_config.json daemon_config.json.backup
cp risk_config.json risk_config.json.backup 2>/dev/null || true

# 备份数据库（如果使用）
cp data/quant.db data/quant.db.backup 2>/dev/null || true

# 备份日志（可选）
tar -czf backup_logs_$(date +%Y%m%d_%H%M%S).tar.gz logs/
```

### 3. 检查磁盘空间

```bash
# 检查可用磁盘空间（至少需要2GB）
df -h

# 清理旧的Docker镜像（可选，释放空间）
docker image prune -a
```

### 4. 检查网络连接

```bash
# 确保可以访问Docker Hub
ping registry-1.docker.io

# 或使用代理（如果需要）
export HTTP_PROXY=http://your-proxy:port
export HTTPS_PROXY=http://your-proxy:port
```

## 🚀 更新步骤

### ⚠️ 重要提示

**如果镜像还没有构建和推送到Docker Hub**，需要先：
1. 构建镜像（参考 [镜像构建与推送指南](镜像构建与推送.md)）
2. 推送到Docker Hub，或
3. 使用本地构建方式（见下方"方式三"）

### 方式一：使用Docker Compose（推荐 - 需要镜像已推送）

#### 步骤1: 停止当前服务

```bash
cd /path/to/Quant_AI_Dashboard

# 停止并删除旧容器
docker compose down

# 确认容器已停止
docker compose ps
```

#### 步骤2: 更新代码和配置

```bash
# 获取最新代码
git pull origin main

# 确保 docker-compose.yml 已更新
cat docker-compose.yml
```

#### 步骤3: 拉取/构建新镜像

```bash
# 拉取所有服务镜像
docker compose pull
```

#### 步骤4: 启动新版本

```bash
# 启动新版本容器
docker compose up -d

# 查看启动状态
docker compose ps
```

#### 步骤5: 查看日志

```bash
# 查看Dashboard日志
docker compose logs -f dashboard

# 查看Daemon日志
docker compose logs -f daemon

# 查看所有服务日志
docker compose logs -f
```

### 方式二：本地构建和使用（镜像未推送时）

如果镜像还没有推送到 Docker Hub，可以在服务器上本地构建：

```bash
# 1. 进入项目目录
cd /path/to/Quant_AI_Dashboard

# 2. 备份数据
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz data/

# 3. 停止当前服务
docker compose down

# 4. 构建并启动新版本
# 这将根据 docker-compose.yml 构建所有服务
docker compose up -d --build
```

### 方式三：手动更新

（不推荐，建议使用 Docker Compose）

## ✅ 验证更新

### 1. 检查容器状态

```bash
# 检查容器是否正常运行
docker compose ps

# 应该看到两个容器都是 "Up" 状态
# dashboard   Up   0.0.0.0:8501->8501/tcp
# daemon      Up
```

### 2. 检查版本信息

```bash
# 方法1: 查看容器镜像版本
docker inspect quant-dashboard | grep -A 5 "Image"

# 方法2: 访问Dashboard查看版本
# 打开浏览器访问 http://your-server:8501
# 查看页面底部或关于页面的版本信息

# 方法3: 在容器内查看
docker exec quant-dashboard python -c "from core.version import get_version; print(get_version())"
# 应该输出: 0.2.0
```

### 3. 功能测试

访问Dashboard并测试以下功能：

- ✅ 登录功能正常
- ✅ 数据加载正常
- ✅ 各页面可以正常访问
- ✅ 后台服务正常运行（检查"后台服务"标签页）

### 4. 检查日志

```bash
# 检查是否有错误日志
docker compose logs dashboard | grep -i error
docker compose logs daemon | grep -i error

# 检查启动日志
docker compose logs dashboard | tail -50
docker compose logs daemon | tail -50
```

## 🔄 回滚方案

如果更新后出现问题，可以快速回滚到旧版本：

### 步骤1: 停止新版本

```bash
docker compose down
```

### 步骤2: 恢复docker-compose.yml

```bash
# 将镜像版本改回 v0.1.4
sed -i 's/v0\.2\.0/v0.1.4/g' docker-compose.yml

# 或手动编辑
vi docker-compose.yml
```

### 步骤3: 启动旧版本

```bash
# 如果本地没有旧镜像，先拉取
docker pull francescoli/quant_ai_dashboard:v0.1.4

# 启动旧版本
docker compose up -d
```

### 步骤4: 恢复数据（如果需要）

```bash
# 如果数据有问题，恢复备份
tar -xzf backup_data_YYYYMMDD_HHMMSS.tar.gz
cp daemon_config.json.backup daemon_config.json
```

## 🔍 常见问题

### 问题1: 镜像拉取失败

**症状**: `docker pull` 命令失败

**解决方案**:
```bash
# 检查网络连接
ping registry-1.docker.io

# 使用国内镜像源（如果在中国）
# 配置Docker镜像加速器
# 编辑 /etc/docker/daemon.json
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}

# 重启Docker服务
sudo systemctl restart docker
```

### 问题2: 容器启动失败

**症状**: `docker compose ps` 显示容器状态为 "Exited"

**解决方案**:
```bash
# 查看详细错误日志
docker compose logs dashboard
docker compose logs daemon

# 检查端口是否被占用
netstat -tulpn | grep 8501

# 检查数据目录权限
ls -la data/
chmod -R 755 data/ logs/
```

### 问题3: 数据丢失

**症状**: 更新后数据不见了

**解决方案**:
```bash
# 检查数据目录挂载
docker inspect quant-dashboard | grep -A 10 Mounts

# 恢复备份数据
tar -xzf backup_data_YYYYMMDD_HHMMSS.tar.gz
```

### 问题4: 配置文件不生效

**症状**: 修改配置后不生效

**解决方案**:
```bash
# 检查配置文件挂载
docker inspect quant-dashboard | grep daemon_config

# 重启容器使配置生效
docker compose restart dashboard daemon
```

### 问题5: 版本号显示不正确

**症状**: Dashboard显示的版本号不是v0.2.0

**解决方案**:
```bash
# 确认镜像版本
docker images | grep quant_ai_dashboard

# 强制重新拉取镜像
docker pull francescoli/quant_ai_dashboard:v0.2.0
docker compose up -d --force-recreate
```
