## 🚀 部署指南

### 1. 本地部署

#### 前置要求

- Python 3.10+
- 至少 2GB 磁盘空间
- 网络连接（用于数据获取）

#### 部署步骤

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp env.example .env
# 编辑 .env 文件

# 3. 启动Dashboard
streamlit run app.py

# 4. 启动API（可选）
python -m uvicorn api.main:app --reload

# 5. 启动后台服务（可选）
python -m core.daemon
```

详细说明请参考 [部署前检查清单](../部署前检查清单.md)

### 2. Docker部署（推荐）

#### 前置要求

- Docker Desktop（Windows/macOS）或 Docker Engine + Docker Compose（Linux）
- 至少 2GB 可用磁盘空间

#### 部署步骤

```bash
# 1. 构建并启动服务
docker compose up -d

# 2. 查看服务状态
docker compose ps

# 3. 查看日志
docker compose logs -f dashboard
docker compose logs -f daemon

# 4. 停止服务
docker compose down
```

详细说明请参考 [DOCKER_DEPLOY.md](../DOCKER_DEPLOY.md)

### 3. 服务器部署

#### 使用systemd（Linux）

创建服务文件 `/etc/systemd/system/quant-dashboard.service`:

```ini
[Unit]
Description=Quant-AI-Dashboard
After=network.target

[Service]
Type=simple
User=quant
WorkingDirectory=/opt/quant-dashboard
Environment="PATH=/opt/quant-dashboard/venv/bin"
ExecStart=/opt/quant-dashboard/venv/bin/streamlit run app.py --server.port=8501
Restart=always

[Install]
WantedBy=multi-user.target
```

启动服务:

```bash
sudo systemctl daemon-reload
sudo systemctl enable quant-dashboard
sudo systemctl start quant-dashboard
sudo systemctl status quant-dashboard
```

## 📊 监控告警

### 1. 系统监控

在Dashboard中访问"系统监控"标签页，可以查看：

- **系统指标**: CPU、内存、磁盘使用率
- **业务指标**: 数据更新延迟、订单执行延迟
- **健康检查**: 数据库、数据源、交易API状态

### 2. 告警配置

#### 邮件告警

```python
from core.monitoring import AlertManager, EmailChannel

alert_manager = AlertManager()
email_channel = EmailChannel(
    smtp_server="smtp.gmail.com",
    smtp_port=587,
    username="your_email@gmail.com",
    password="your_password",
    to_email="alerts@example.com"
)
alert_manager.add_channel("email", email_channel)
```

#### Webhook告警

```python
from core.monitoring import AlertManager, WebhookChannel

webhook_channel = WebhookChannel(
    webhook_url="https://your-webhook-url.com/alerts"
)
alert_manager.add_channel("webhook", webhook_channel)
```

#### 告警规则

```python
alert_manager.add_alert_rule(
    name="CPU告警",
    metric_name="cpu_usage",
    threshold=80.0,
    comparison=ComparisonOperator.GT,
    severity=AlertSeverity.WARNING,
    channels=["email", "webhook"]

```

详细说明请参考 [监控系统使用说明](../core/监控系统使用说明.md)

### 3. 监控指标

系统会收集以下指标：

- **系统指标**: CPU、内存、磁盘、网络

## 📝 日志管理

### 1. 日志位置

- **Dashboard日志**: `logs/daemon.log`
- **API日志**: 控制台输出（可配置为文件）
- **审计日志**: `logs/audit/audit_YYYYMMDD.log`

### 2. 日志级别

- **DEBUG**: 调试信息
- **INFO**: 一般信息
- **WARNING**: 警告信息
- **ERROR**: 错误信息
- **CRITICAL**: 严重错误

### 3. 日志轮转

#### 使用logrotate（Linux）

创建 `/etc/logrotate.d/quant-dashboard`:

```
/path/to/quant-dashboard/logs/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 quant quant
}
```

#### 手动清理

```bash
# 清理7天前的日志
find logs/ -name "*.log" -mtime +7 -delete

# 压缩旧日志
find logs/ -name "*.log" -mtime +1 -exec gzip {} \;
```

## 💾 备份恢复

### 1. 数据备份

#### 手动备份

```bash
# 备份数据目录
tar -czf backup_$(date +%Y%m%d).tar.gz data/

# 备份数据库
cp data/quant.db backup/quant_$(date +%Y%m%d).db

# 备份配置文件
cp daemon_config.json backup/
cp risk_config.json backup/
```

#### 自动备份脚本

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR="backup/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

# 备份数据
tar -czf $BACKUP_DIR/data.tar.gz data/

# 备份配置
cp daemon_config.json $BACKUP_DIR/
cp risk_config.json $BACKUP_DIR/

# 备份数据库
cp data/quant.db $BACKUP_DIR/

# 清理7天前的备份
find backup/ -type d -mtime +7 -exec rm -rf {} \;
```

#### 使用cron定时备份

```bash
# 编辑crontab
crontab -e

# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh
```

### 2. 数据恢复

```bash
# 恢复数据
tar -xzf backup_20250101.tar.gz

# 恢复数据库
cp backup/quant_20250101.db data/quant.db

# 恢复配置
cp backup/daemon_config.json .
cp backup/risk_config.json .
```

## 🔧 故障排查

### 1. Dashboard无法启动

#### 检查端口占用

```bash
# Linux/macOS
netstat -an | grep 8501
lsof -i :8501

# Windows
netstat -an | findstr 8501
```

#### 检查日志

```bash
# 查看Dashboard日志
tail -f logs/daemon.log

# 查看错误信息
streamlit run app.py --logger.level=debug
```

#### 常见问题

- **端口被占用**: 修改 `app.py` 中的端口号或停止占用端口的进程
- **依赖缺失**: 运行 `pip install -r requirements.txt`
- **配置错误**: 检查 `.env` 文件配置

### 2. 数据获取失败

#### 检查网络连接

```bash
# 测试网络连接
ping api.example.com

# 测试API访问
curl https://api.example.com/health
```

#### 检查API密钥

- 检查 `.env` 文件中的API密钥是否正确
- 检查API密钥是否过期
- 检查API调用频率限制

#### 检查数据源

```python
# 在Python中测试数据源
import akshare as ak
data = ak.stock_zh_a_hist(symbol="000001", period="daily")
print(data.head())
```

### 3. 模型训练失败

#### 检查数据

- 确保有足够的历史数据（至少60天）
- 检查数据质量（无缺失值、异常值）

#### 检查资源

```bash
# 检查内存使用
free -h  # Linux
vm_stat  # macOS

# 检查磁盘空间
df -h
```

#### 查看训练日志

```bash
# 查看训练日志
tail -f logs/daemon.log | grep training
```

### 4. 交易执行失败

#### 检查风险限制

- 检查是否超过仓位限制
- 检查是否超过损失限制
- 检查是否触发紧急停止

#### 检查订单状态

```python
from core.order_manager import OrderManager

order_manager = OrderManager()
status = order_manager.get_order_status(order_id)
print(status)
```

### 5. 性能问题

#### 检查系统资源

```bash
# 查看CPU和内存使用
top  # Linux/macOS
taskmgr  # Windows

# 查看磁盘IO
iostat -x 1  # Linux
```

#### 优化建议

- 增加缓存大小
- 优化数据库查询
- 使用异步处理
- 增加服务器资源

## 🔒 安全建议

### 1. 密码安全

- 使用强密码
- 使用密码哈希（`APP_LOGIN_PASSWORD_HASH`）
- 定期更换密码

### 2. API密钥安全

- 不要将API密钥提交到Git
- 使用环境变量管理密钥
- 定期轮换API密钥

### 3. 网络安全

- 生产环境使用HTTPS
- 配置防火墙规则
- 限制访问IP（如需要）

### 4. 数据安全

- 定期备份数据
- 加密敏感数据
- 设置文件权限
