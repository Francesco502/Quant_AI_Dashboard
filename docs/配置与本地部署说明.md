# 配置与本地部署说明

本文说明如何配置项目环境变量，以及本地部署时前后端的配置与启动方式。

---

## 一、配置方式概览

1. **后端**：在项目根目录使用 `.env` 文件（由 `env.example` 复制并填值）。
2. **前端**：通过环境变量 `NEXT_PUBLIC_API_URL` 指定后端 API 地址；本地开发时可在启动前设置或写进 `.env.local`。

---

## 二、后端配置（.env）

### 2.1 创建 .env

在项目根目录（与 `api/`、`core/` 同级）执行：

```bash
# Windows PowerShell
Copy-Item env.example .env

# 或手动复制 env.example 为 .env，再用编辑器填写
```

编辑 `.env`，按需取消注释并填入实际值。

### 2.2 必选 / 推荐配置

| 变量 | 说明 | 本地开发建议 |
|------|------|--------------|
| `SECRET_KEY` | JWT 与会话加密密钥，生产必须改 | 开发可沿用示例值；生产用 `python -c "import secrets; print(secrets.token_urlsafe(32))"` 生成 |
| `APP_LOGIN_PASSWORD` 或 `APP_LOGIN_PASSWORD_HASH` | 登录密码 | 二选一，开发可设 `APP_LOGIN_PASSWORD=你的密码` |

### 2.3 可选：LLM 决策分析

未配置时使用占位客户端（返回占位结果，不报错）。若要真实调用 LLM，**至少配置一种**：

| 变量 | 说明 | 示例 |
|------|------|------|
| `OPENAI_API_KEY` | OpenAI 或兼容接口的 API Key | 必填 |
| `OPENAI_BASE_URL` | 兼容接口地址（DeepSeek/通义等） | `https://api.deepseek.com/v1` |
| `OPENAI_MODEL` | 模型名 | `deepseek-chat`、`gpt-4o-mini` 等 |
| `GEMINI_API_KEY` | Google Gemini Key | 与上面二选一或同时存在时按代码优先级 |
| `GEMINI_MODEL` | **Gemini 模型名**（服务端默认） | 如 `gemini-1.5-pro`、`gemini-1.5-flash`、`gemini-2.0-flash`；不填则代码默认 `gemini-1.5-pro` |
| `ANTHROPIC_API_KEY` / `ANTHROPIC_MODEL` | Claude | 可选 |

自选股（定时/run-daily 用）：

- `DAILY_ANALYSIS_TICKERS=600519,000858,hk00700,AAPL`（逗号分隔）

**模型可在前端覆盖**：决策仪表盘页提供「模型（可选）」输入框，留空则使用上述服务端默认（`.env` 中的 `GEMINI_MODEL` 或 `OPENAI_MODEL`）；填写则本次请求使用该模型（如 `gemini-1.5-flash`）。

### 2.4 可选：数据源、推送、新闻

- **数据源**：`TUSHARE_TOKEN`、`ALPHA_VANTAGE_KEY` 等按需填写。
- **推送**：企微/飞书/钉钉/Telegram/邮件等，见 `env.example` 中 `WECHAT_WEBHOOK_URL`、`FEISHU_WEBHOOK_URL` 等。
- **新闻搜索**：`TAVILY_API_KEYS`、`NEWS_MAX_AGE_DAYS=3` 等，见 `core/search_service` 与 `env.example`。

### 2.5 可选：CORS（生产部署时重要）

- `CORS_ORIGINS=https://你的前端域名`，多个用逗号分隔。本地前后端同机可先不设或留空。

## 三、本地部署时如何配置

### 3.1 后端

1. 在项目根目录确保已有 `.env`（可从 `env.example` 复制）。
2. 按第二节填写至少 `SECRET_KEY` 和登录密码；需要真实 LLM 时配置 `OPENAI_*` 或 `GEMINI_*`。
3. 安装依赖并启动：

```bash
pip install -r requirements.txt
# 启动（默认 8685 端口）
py -m uvicorn api.main:app --host 0.0.0.0 --port 8685
```

4. 验证：浏览器或 `curl` 访问 `http://localhost:8685/api/health`，返回 200 即表示后端正常。

### 3.2 前端

前端需要知道后端 API 的地址，通过 **`NEXT_PUBLIC_API_URL`** 指定。

**方式 A：启动前临时设置（适合快速本地试跑）**

```powershell
# Windows PowerShell，在 web 目录下
cd web
$env:NEXT_PUBLIC_API_URL="http://localhost:8685/api"
npm run dev
```

```bash
# Linux / macOS
cd web
export NEXT_PUBLIC_API_URL=http://localhost:8685/api
npm run dev
```

**方式 B：使用 .env.local（推荐，不用每次敲命令）**

在 `web/` 目录下新建 `.env.local`（不要提交到 git），内容示例：

```bash
# web/.env.local
NEXT_PUBLIC_API_URL=http://localhost:8685/api
```

然后直接执行：

```bash
cd web
npm install
npm run dev
```

Next.js 会自动加载 `.env.local`，前端请求会发往 `http://localhost:8685/api`。

### 3.3 本地部署检查清单

| 步骤 | 说明 |
|------|------|
| 1. 根目录有 `.env` | 从 `env.example` 复制，填好 `SECRET_KEY`、登录密码等 |
| 2. 后端端口一致 | 默认 8685；若改端口，前端 `NEXT_PUBLIC_API_URL` 要一起改 |
| 3. 前端 API 地址 | 设 `NEXT_PUBLIC_API_URL=http://localhost:8685/api`（或你的后端地址） |
| 4. 先启后端再启前端 | 先 `uvicorn ... --port 8685`，再 `npm run dev` |

### 3.4 若后端不是本机（例如连远程或另一台电脑）

- 将 `NEXT_PUBLIC_API_URL` 设为实际后端地址，例如：  
  `http://192.168.1.100:8685/api` 或 `https://your-api.example.com/api`。
- 后端 `.env` 中 `CORS_ORIGINS` 需包含前端访问的域名或 `http://localhost:3000`（Next 默认端口），否则浏览器会拦请求。

## 四、快速命令汇总（本地）

```powershell
# 1. 后端（项目根目录）
py -m uvicorn api.main:app --host 0.0.0.0 --port 8685

# 2. 前端（新开终端，web 目录）
cd web
$env:NEXT_PUBLIC_API_URL="http://localhost:8685/api"
npm run dev

# 3. 验证后端
curl http://localhost:8685/api/health
```

前端默认打开 `http://localhost:3000`，登录后即可使用；决策分析等需 LLM 时请在 `.env` 中配置对应 Key。
