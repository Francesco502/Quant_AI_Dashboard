## ✅ 前置要求

### 1. Docker环境

确保已安装Docker：
```bash
docker --version
docker-compose --version
```

### 2. Docker Hub账号

1. 注册Docker Hub账号：https://hub.docker.com/
2. 登录Docker Hub：
```bash
docker login
# 输入用户名和密码
```

### 3. 镜像仓库名称

确认你的 docker-compose.yml 中定义的镜像名称。通常包含：
- 后端服务：`francescoli/quant_backend`
- 前端服务：`francescoli/quant_frontend`

## 🔨 构建镜像

推荐使用 Docker Compose 进行构建，它可以同时处理前后端服务的镜像构建。

### 方式一：使用 Docker Compose (推荐)

```bash
# 进入项目目录
cd Quant_AI_Dashboard

# 构建所有服务镜像
docker compose build

# 如果需要无缓存构建（强制更新）
docker compose build --no-cache
```

### 方式二：手动构建单独镜像

如果需要单独构建服务镜像：

```bash
# 构建后端镜像
docker build -t francescoli/quant_backend:latest .

# 构建前端镜像
cd web
docker build -t francescoli/quant_frontend:latest .
```

## 📤 推送镜像

### 1. 登录Docker Hub

```bash
docker login
# 输入用户名和密码
```

### 2. 推送所有服务镜像

```bash
docker compose push
```

### 3. 验证推送

在 Docker Hub 网页上查看对应的仓库 tags。

## 🏠 本地部署（不推送）

如果不想推送到 Docker Hub，可以在本地构建和使用：

```bash
# 构建并启动所有服务
docker compose up -d --build
```

## ✅ 验证镜像

### 1. 检查镜像

```bash
# 查看本地镜像
docker images | grep quant
```

### 2. 检查服务状态

```bash
docker compose ps
```

### 3. 测试完整功能

```bash
# 启动测试容器
docker run -d \
  --name test-dashboard \
  -p 8502:8501 \
  francescoli/quant_ai_dashboard:v1.0.0

# 访问测试
# http://localhost:8502

# 清理测试容器
docker stop test-dashboard
docker rm test-dashboard
```

## 🔄 完整流程示例

### 场景1：构建并推送到Docker Hub

```bash
# 1. 登录Docker Hub
docker login

# 2. 构建镜像
docker build -t francescoli/quant_ai_dashboard:v1.0.0 \
             -t francescoli/quant_ai_dashboard:latest .

# 3. 验证构建
docker images | grep quant_ai_dashboard

# 4. 推送镜像
docker push francescoli/quant_ai_dashboard:v1.0.0
docker push francescoli/quant_ai_dashboard:latest

# 5. 在服务器上拉取
# docker pull francescoli/quant_ai_dashboard:v1.0.0
```

### 场景2：本地构建和使用

```bash
# 1. 构建本地镜像
docker build -t quant_ai_dashboard:v1.0.0 .

# 2. 修改docker-compose.yml使用本地镜像
# image: quant_ai_dashboard:v1.0.0

# 3. 启动服务
docker compose up -d
```

### 场景3：导出和导入镜像

如果服务器无法访问Docker Hub，可以导出镜像文件：

```bash
# 在构建机器上导出镜像
docker save francescoli/quant_ai_dashboard:v1.0.0 \
  -o quant_ai_dashboard_v1.0.0.tar

# 传输到服务器（使用scp、ftp等）
scp quant_ai_dashboard_v1.0.0.tar user@server:/path/to/

# 在服务器上导入镜像
docker load -i quant_ai_dashboard_v1.0.0.tar

# 修改docker-compose.yml使用导入的镜像
# image: francescoli/quant_ai_dashboard:v1.0.0
```

## ⚠️ 常见问题

### 问题1: 构建失败 - 依赖安装超时

**解决方案**:
```bash
# 增加超时时间和重试次数
docker build --build-arg PIP_TIMEOUT=600 \
             --build-arg PIP_RETRIES=10 \
             -t francescoli/quant_ai_dashboard:v1.0.0 .
```

### 问题2: 推送失败 - 认证错误

**解决方案**:
```bash
# 重新登录
docker logout
docker login

# 检查镜像标签是否正确
docker images | grep quant_ai_dashboard
```

### 问题3: 镜像太大

**解决方案**:
- 使用多阶段构建
- 优化.dockerignore
- 清理不必要的文件
- 使用alpine基础镜像

### 问题4: 构建缓存问题

**解决方案**:
```bash
# 不使用缓存重新构建
docker build --no-cache -t francescoli/quant_ai_dashboard:v1.0.0 .
```
