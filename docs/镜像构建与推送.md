## ✅ 前置要求

### 1. Docker环境

确保已安装Docker：
```bash
docker --version
docker-compose --version
```

### 2. Docker Hub账号

1. 注册Docker Hub账号：https://hub.docker.com/
2. 登录Docker Hub：
```bash
docker login
# 输入用户名和密码
```

### 3. 镜像仓库名称

确认你的Docker Hub用户名和仓库名称：
- 用户名：`francescoli`（或你的用户名）
- 仓库名：`quant_ai_dashboard`
- 完整镜像名：`francescoli/quant_ai_dashboard:v0.2.0`

## 🔨 构建镜像

### 方式一：构建并标记版本

```bash
# 进入项目目录
cd Quant_AI_Dashboard

# 构建镜像并标记版本
docker build -t francescoli/quant_ai_dashboard:v0.2.0 .

# 同时标记为latest（可选）
docker build -t francescoli/quant_ai_dashboard:v0.2.0 \
             -t francescoli/quant_ai_dashboard:latest .
```

### 方式二：使用构建脚本

创建 `build_image.sh`（Linux/macOS）或 `build_image.bat`（Windows）：

```bash
#!/bin/bash
# build_image.sh

VERSION="v0.2.0"
IMAGE_NAME="francescoli/quant_ai_dashboard"

echo "构建镜像: ${IMAGE_NAME}:${VERSION}"

docker build -t ${IMAGE_NAME}:${VERSION} \
             -t ${IMAGE_NAME}:latest \
             .

echo "构建完成！"
echo "镜像列表:"
docker images | grep ${IMAGE_NAME}
```

### 构建参数说明

```bash
docker build \
  -t francescoli/quant_ai_dashboard:v0.2.0 \  # 镜像标签
  --build-arg BUILD_DATE=$(date -u +'%Y-%m-%dT%H:%M:%SZ') \  # 构建日期（可选）
  --build-arg VERSION=v0.2.0 \  # 版本号（可选）
  .
```

### 构建优化

如果构建速度慢，可以使用多阶段构建或缓存：

```bash
# 使用缓存加速构建
docker build --cache-from francescoli/quant_ai_dashboard:v0.1.4 \
             -t francescoli/quant_ai_dashboard:v0.2.0 .
```

## 📤 推送镜像

### 1. 登录Docker Hub

```bash
docker login
# 输入用户名和密码
```

### 2. 推送指定版本

```bash
# 推送v0.2.0版本
docker push francescoli/quant_ai_dashboard:v0.2.0

# 推送latest标签（如果标记了）
docker push francescoli/quant_ai_dashboard:latest
```

### 3. 验证推送

在Docker Hub网页上查看：
https://hub.docker.com/r/francescoli/quant_ai_dashboard/tags

或使用命令行：
```bash
# 查看远程镜像信息（需要Docker Hub API）
curl -s https://hub.docker.com/v2/repositories/francescoli/quant_ai_dashboard/tags/ | grep v0.2.0
```

## 🏠 本地部署（不推送）

如果不想推送到Docker Hub，可以在本地构建和使用：

### 1. 本地构建

```bash
# 构建镜像（使用本地标签）
docker build -t quant_ai_dashboard:v0.2.0 .
```

### 2. 修改docker-compose.yml

将镜像名称改为本地标签：

```yaml
services:
  dashboard:
    image: quant_ai_dashboard:v0.2.0  # 使用本地镜像
    # ...
  daemon:
    image: quant_ai_dashboard:v0.2.0  # 使用本地镜像
    # ...
```

### 3. 启动服务

```bash
docker compose up -d
```

## ✅ 验证镜像

### 1. 检查镜像

```bash
# 查看本地镜像
docker images | grep quant_ai_dashboard

# 查看镜像详细信息
docker inspect francescoli/quant_ai_dashboard:v0.2.0
```

### 2. 测试运行

```bash
# 测试运行容器
docker run --rm francescoli/quant_ai_dashboard:v0.2.0 \
  python -c "from core.version import get_version; print(get_version())"

# 应该输出: 0.2.0
```

### 3. 测试完整功能

```bash
# 启动测试容器
docker run -d \
  --name test-dashboard \
  -p 8502:8501 \
  francescoli/quant_ai_dashboard:v0.2.0

# 访问测试
# http://localhost:8502

# 清理测试容器
docker stop test-dashboard
docker rm test-dashboard
```

## 🔄 完整流程示例

### 场景1：构建并推送到Docker Hub

```bash
# 1. 登录Docker Hub
docker login

# 2. 构建镜像
docker build -t francescoli/quant_ai_dashboard:v0.2.0 \
             -t francescoli/quant_ai_dashboard:latest .

# 3. 验证构建
docker images | grep quant_ai_dashboard

# 4. 推送镜像
docker push francescoli/quant_ai_dashboard:v0.2.0
docker push francescoli/quant_ai_dashboard:latest

# 5. 在服务器上拉取
# docker pull francescoli/quant_ai_dashboard:v0.2.0
```

### 场景2：本地构建和使用

```bash
# 1. 构建本地镜像
docker build -t quant_ai_dashboard:v0.2.0 .

# 2. 修改docker-compose.yml使用本地镜像
# image: quant_ai_dashboard:v0.2.0

# 3. 启动服务
docker compose up -d
```

### 场景3：导出和导入镜像

如果服务器无法访问Docker Hub，可以导出镜像文件：

```bash
# 在构建机器上导出镜像
docker save francescoli/quant_ai_dashboard:v0.2.0 \
  -o quant_ai_dashboard_v0.2.0.tar

# 传输到服务器（使用scp、ftp等）
scp quant_ai_dashboard_v0.2.0.tar user@server:/path/to/

# 在服务器上导入镜像
docker load -i quant_ai_dashboard_v0.2.0.tar

# 修改docker-compose.yml使用导入的镜像
# image: francescoli/quant_ai_dashboard:v0.2.0
```

## ⚠️ 常见问题

### 问题1: 构建失败 - 依赖安装超时

**解决方案**:
```bash
# 增加超时时间和重试次数
docker build --build-arg PIP_TIMEOUT=600 \
             --build-arg PIP_RETRIES=10 \
             -t francescoli/quant_ai_dashboard:v0.2.0 .
```

### 问题2: 推送失败 - 认证错误

**解决方案**:
```bash
# 重新登录
docker logout
docker login

# 检查镜像标签是否正确
docker images | grep quant_ai_dashboard
```

### 问题3: 镜像太大

**解决方案**:
- 使用多阶段构建
- 优化.dockerignore
- 清理不必要的文件
- 使用alpine基础镜像

### 问题4: 构建缓存问题

**解决方案**:
```bash
# 不使用缓存重新构建
docker build --no-cache -t francescoli/quant_ai_dashboard:v0.2.0 .
```
